## 一、公众号内容自动化技术方案对比

### 方案总览

| 方案 | 技术栈 | 成本 | 稳定性 | 适合场景 |
|------|--------|------|--------|---------|
| **微信官方 API** | Python/Node + API | 认证费 ¥300/年 | ⭐⭐⭐⭐⭐ | 已认证服务号 |
| **CDP + 浏览器** | Python + Chrome CDP | 免费 | ⭐⭐⭐⭐ | 个人号/订阅号（我们的方案） |
| **n8n / Dify 工作流** | 可视化平台 | $20-50/月 | ⭐⭐⭐ | 非技术人员 |
| **RPA 工具** | 影刀/UiBot | ¥100-500/月 | ⭐⭐⭐ | 简单重复操作 |
| **WeRoBot 框架** | Python | 免费 | ⭐⭐⭐⭐ | 消息自动回复 |

### 我们的方案详解

```
Chrome 浏览器（已登录公众平台）
    ↓ CDP 协议
CDP Proxy（localhost:3456）
    ↓ HTTP API
Python 脚本（同步 XHR 注入）
    ↓
公众平台内部 API
```

**优势：**
- 不需要服务号认证（个人订阅号即可）
- 完全利用浏览器登录态，无需处理 OAuth
- 可以调用所有后台功能（包括草稿箱、素材管理等）

**劣势：**
- 依赖本地 Chrome 浏览器运行
- 登录态有过期风险（通常 7 天）
- 不适合部署到服务器

## 二、AI 内容合规要求（2025年起）

### 《人工智能生成合成内容标识办法》

**2025 年 3 月 1 日正式实施**，由国家互联网信息办公室发布。

**核心要求：**

1. **强制标注**：所有 AI 生成/辅助生成的内容必须添加标识
2. **标注范围**：文字、图片、音频、视频、虚拟场景
3. **标注位置**：显著位置，不得隐藏或难以辨认
4. **平台责任**：平台需要提供标注工具和检测能力

**微信公众号的具体执行：**

- 2025 年 Q1 起，公众号后台新增"AI 生成内容标注"选项
- 发布时可勾选"本文包含 AI 生成内容"
- 未标注被举报后可能面临：限流、警告、封号
- 目前执法尚在初期，但趋势明确

**我们的应对策略：**

```html
<!-- 在文章末尾添加 -->
<p style="color: #999; font-size: 12px; margin-top: 30px; padding-top: 15px; border-top: 1px solid #eee;">
  📌 本文由 AI 辅助采集和整理，经人工审核后发布。
</p>
```

**建议措辞**（不同场景）：
- 纯 AI 采集+摘要："本文由 AI 辅助整理，人工审核发布"
- AI 写评论/点评："本文包含 AI 生成的编辑点评"
- AI 生成封面图："封面图由 AI 生成"

## 三、公众号发布最佳时间

### 数据分析（2024-2025 公开数据）

| 时间段 | 特点 | 打开率 | 适合内容 |
|--------|------|--------|---------|
| **6:30-8:00** | 早起高峰 | ⭐⭐⭐⭐ | 早报、日报 ✅ |
| **8:00-9:00** | 通勤高峰 | ⭐⭐⭐⭐⭐ | 资讯速览 ✅ |
| **11:30-13:00** | 午休 | ⭐⭐⭐⭐ | 轻阅读 |
| **17:00-18:30** | 下班通勤 | ⭐⭐⭐ | 深度内容 |
| **20:00-22:00** | 晚间黄金 | ⭐⭐⭐⭐⭐ | 所有类型 |
| **22:00-23:30** | 睡前 | ⭐⭐⭐ | 轻松内容 |

### 对 AI 日报的建议

**推荐发布时间：早上 7:00-7:30**

理由：
1. AI 日报是"资讯类"内容，用户希望"开始工作前了解最新动态"
2. 早间竞争较少（大部分公众号晚间发布）
3. 形成用户习惯："每天早上看 XX 的 AI 日报"
4. 海外 Newsletter 多在早晨 6-8 点发送（已验证的时间窗口）

**当前状态：**
- launchd 设置为 9:57 触发
- 建议改为 6:30 触发采集，7:00 前发布

## 四、排版工具和方案

### 微信公众号排版限制

**微信编辑器的技术限制：**
- ❌ 不支持 `<style>` 标签
- ❌ 不支持 class/id 选择器
- ❌ 不支持外部 CSS
- ❌ 不支持 `<link>` 标签
- ❌ 不支持 JavaScript
- ❌ 不支持 CSS 变量、animation、grid
- ✅ 支持内联 style（`style="..."`）
- ✅ 支持基本 HTML 标签（div/p/span/table/img/a）
- ✅ 支持有限的 CSS 属性（color/font/margin/padding/border/background-color）

### 排版工具对比

| 工具 | 类型 | 优势 | 劣势 |
|------|------|------|------|
| **mdnice** | 在线工具 | 主题丰富，一键复制 | 免费版限制 |
| **Doocs MD** | 开源(8k star) | 可自建，完全免费 | 需要自己部署 |
| **wechat-format** | 开源 | 极简轻量 | 功能少 |
| **135 编辑器** | 商业工具 | 模板多 | 付费 ¥99/年起 |
| **秀米** | 商业工具 | 可视化排版 | 付费 |
| **自研模板** | 自建 | 完全定制 | 开发成本 |

### 我们的选择：自研 HTML 模板

**理由：**
1. 需要自动化生成，不能依赖在线编辑器
2. 模板固定后几乎不需要改动
3. 完全掌控样式和结构

**核心原理：**
```
Markdown/JSON 数据
    ↓ Python 模板引擎
HTML（全部 inline style）
    ↓ CDP 提交
公众号草稿箱
```

## 五、公众号增长策略

### 阶段一：冷启动（0-500 粉丝）

**关键动作：**
1. **朋友圈首发**：让朋友转发，预计触达 500-1000 人
2. **微信群分享**：加入 3-5 个 AI 相关微信群
3. **知乎回答引流**：回答 AI 相关问题，文末引导关注
4. **小红书同步**：将日报内容制作成图文笔记
5. **坚持日更**：至少 30 天不断更

**预期时间**：1-2 个月

### 阶段二：小步快跑（500-2000 粉丝）

**关键动作：**
1. **开通流量主**（500 粉即可）
2. **互推**：找同量级 AI 类公众号互推
3. **内容升级**：加入"编辑点评"增加个人风格
4. **搜一搜优化**：标题包含热门搜索词
5. **固定栏目**：如"每周 AI 工具推荐"增加粘性

**预期时间**：2-4 个月

### 阶段三：加速期（2000-10000 粉丝）

**关键动作：**
1. **建立社群**：创建读者微信群
2. **视频号联动**：把日报做成 1 分钟短视频
3. **知识星球上线**：提供深度内容和社区
4. **商务合作**：接 AI 工具推广
5. **内容矩阵**：日报 + 周报 + 工具评测

### 阶段四：稳定期（10000+ 粉丝）

**关键动作：**
1. **品牌广告**：互选广告平台
2. **课程/培训**：AI 工具使用教程
3. **企业服务**：定制行业简报
4. **产品化**：将自动化采集系统 SaaS 化

## 六、自动化工作流最佳实践

### 完整自动化链路

```
定时触发（launchd/cron）
    ↓
检查环境（CDP Proxy/登录态）
    ↓
数据采集（公众号文章）
    ↓
数据处理（标准化/去重/打标签）
    ↓
内容生成（bundle/摘要/点评）
    ↓
输出生成（HTML/公众号稿/封面图）
    ↓
发布（草稿箱/GitHub Pages）
    ↓
通知（成功/失败）
    ↓
日志归档
```

### 监控和告警

**建议监控点：**

| 检查项 | 频率 | 告警方式 |
|--------|------|---------|
| CDP Proxy 可达 | 每次运行前 | macOS 通知 |
| 登录态有效 | 每次运行前 | macOS 通知 |
| 采集数量 > 0 | 每次运行后 | macOS 通知 |
| 去重后数量 | 每次运行后 | 日志记录 |
| 草稿提交成功 | 每次运行后 | macOS 通知 |
| 日志文件大小 | 每周 | 自动轮转 |

### 容错策略

1. **采集失败**：跳过当前源，继续下一个
2. **无新文章**：记录但不生成空 bundle
3. **去重异常**：降级为简单 URL 去重
4. **AI API 超时**：使用缓存或降级为无点评模式
5. **草稿提交失败**：保留 HTML 文件，人工提交
6. **Git 推送失败**：本地保留，下次重试

## 七、竞品自动化方案参考

### RSS-GPT（开源）

- **地址**：github.com/yinan-c/RSS-GPT
- **方案**：RSS 订阅 + GPT 生成摘要 + GitHub Actions 定时运行
- **优势**：完全免费（GitHub Actions 额度内）、无需服务器
- **劣势**：不支持微信公众号源
- **借鉴**：GitHub Actions 定时执行的思路

### NewLetter（海外工具）

- **方案**：AI 自动策展 + 邮件发送
- **特色**：可以从多个源自动聚合内容
- **借鉴**：多源聚合的 UI 和体验设计

### Feedful

- **方案**：RSS + AI 筛选 + Newsletter 生成
- **特色**：可视化规则配置（筛选条件）
- **借鉴**：内容筛选规则的设计
