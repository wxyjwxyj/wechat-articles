# 大模型预训练数据工程前置知识

> 写给完全零基础的读者——读完这篇，你就能看懂横纵分析报告的主体内容。

---

## 1. 模型的"食物"长什么样？

大模型不是生下来就聪明的。它的"学习方式"很简单：**读一段文本，猜下一个词是什么，猜错了就调整自己的参数，让下次猜对**。重复几千亿次，它就学会了语言、知识、推理——这就是预训练。

所以训练数据就是模型的"食物"。那模型"吃"的是什么？不是维基百科那种整洁的文章，而是互联网原始网页的文本提取物。举个例子，假设模型读到这样一段"原始食物"：

> **为了获得最佳浏览体验 请 点击 这里 跳过 广告 热门推荐 限时抢购 ￥9.9 立即购买 月销10万+ 用户评论 好好用啊哈哈哈哈哈哈 猜你喜欢 更多优惠 点击查看**

这就是 CommonCrawl 里随处可见的页面——电商网站产品页。模型如果读了大量这种东西，它学会的是"从关键词堆砌到点击诱饵的预测模式"，而不是推理。

## 2. 那好的"食物"长什么样？

> **1987年，中国GDP首次突破1万亿元人民币。这一数字背后，是改革开放政策的持续推进和农村经济体制改革的显著成效。从计划经济到市场经济的转轨过程中，中国农业劳动生产率提高了近40%。**

模型读这种数据，学到的是因果关系（"背后是……"）、时间线（"1987年"）、逻辑衔接（"从……到……"）。质量立判。

**数据工程要解决的根本问题就是：给你一个亿万个网页组成的数据海洋，怎么从中捞出那些"教科书级"的好数据，扔掉那些像电商评论一样的垃圾？**

## 3. 数据规模的直觉

GPT-4 的训练数据量 OpenAI 未正式公开，社区估算约 13 万亿 token（基于 Chinchilla 法则反推：1.8T 参数 × 约 7-8 倍数据配比）。13 万亿是什么概念？全中国 14 亿人每人写 1 万字，才 1.4 万亿字——差了近 10 倍。这就是为什么数据工程的第一步是"量大管饱"。

但 CommonCrawl 每个月爬取的网页原始文本，经过清洗后可能只剩 10%-30%。**90% 的互联网内容都是噪音。**

## 4. 去重的直觉：为什么复制粘贴是毒药？

如果一篇 Wikipedia 文章被 1000 个博客原封不动转载，模型训练时会"读到"这篇 1000 次。结果是：模型背会了这篇文章，但没有理解其中的知识。测试时问你"光合作用的原理"，它把 Wikipedia 原文一字不差背出来——这不是智能，是死记硬背。

更糟糕的是，这种重复会让模型丧失泛化能力：遇到没见过的问题时，它更倾向于匹配训练集中见过的文本模式，而不是推理。所以去重——移除内容相同的重复文档——是数据工程中最被低估但效果最显著的环节。

## 5. 数据质量 vs 数据数量

你可以把数据质量想象成炒菜的食材。你有一万吨普通土豆（低质量大数据），和一吨顶级松露（高质量小数据）。做出来的菜哪个好吃？不一定——土豆量大但上限低，松露量少但味道浓。

数据工程在 2024 年的核心发现就是这个：2.6T 精心筛选的 token 训练的模型，效果接近 15T 未精细筛选的 token。**数据的"信息密度"比"信息总量"更重要。** 读 1 本经典教科书，比读 100 本三流博客学到更多——这句话在人类世界成立，在 AI 世界也成立。

## 6. 代码数据为什么重要？

2022-2023 年的模型配比中，代码数据只占 4-5%。2024 年的模型普遍提到 17% 以上。为什么？

因为代码跟自然语言有一个根本性区别：**代码要么能运行要么不能。** 网页文本里说"1+1=3"没人会报错，但代码写 `x = 1 + 1; assert x == 3` 会报错。代码数据的"事实性检查"是天然的——这使得模型在代码数据上学到的逻辑推理能力比自然语言强得多。一个在 17% 代码上训练的模型，其数学推理能力可能超过一个在 5% 代码上训练的大得多的模型。

## 7. 数据配比是什么？

想象你在给模型设计课程表：

- **网页数据** = 通识教育课（量最大，什么都覆盖，但不深入）
- **书籍数据** = 经典阅读课（深度好，但覆盖窄）
- **代码数据** = 编程训练课（逻辑推理强）
- **学术论文** = 专业课（前沿但难度高）

不同公司（不同学校）的课程表（数据配比）不同。LLaMA 3 的"课程表"是网页 50%、代码 17%、数学推理 25%、多语言 8%。Qwen 的课程表是 18T 的大而全。**配比决定了模型擅长什么——你的时间花在哪里，你就擅长什么，对模型也一样。**

## 8. 数据清洗 Pipeline 长什么样？

从工程角度看，数据工程可以抽象为三层：

```
采集层 → 清洗层 → 配比层
```

- **采集层**：从 CommonCrawl、书籍、代码仓库、学术论文等渠道获取原始数据
- **清洗层**：去噪、去重、语言过滤、质量筛选
- **配比层**：决定不同数据源的混合比例

### 实际的数据清洗流水线

```
CommonCrawl 原始数据（包含所有网页的 HTML）
  ↓ 文本提取（去掉 HTML 标签，只留纯文字）
  ↓ 语言检测（只保留目标语言）
  ↓ 启发式过滤（太短不要、符号太多的不要、脏词不要）
  ↓ 去重（URL 去重 + 内容去重）
  ↓ 质量过滤（分类器或模型评分）
  ↓ 最终语料
```

每个环节都在移除"垃圾"，但每一步也有误删好数据的风险。

### 最早的系统化方案：C4（Colossal Clean Crawled Corpus）和 DCLM（DataComp for Language Models）

2019 年 Google 提出了 **C4（Colossal Clean Crawled Corpus）**，是第一套系统化的数据清洗规则集：移除脏词、过多标点、过短文档、重复内容。今天所有数据清洗工作仍以 C4 作为基线。

2024 年的 **DCLM** 更进一步：不是用人定规则，而是用模型本身来筛选数据。它训练了一个分类器给每个文档评分，只保留高分文档。效果惊人——用 2.6T 精心筛选的 token 训练的 7B 模型，接近用 15T token 训练的 Llama 3 8B。**数据质量的差距相当于 6 倍的数据量差距。**

### 困惑度（Perplexity）在清洗中的作用

困惑度是衡量模型对文本"预测难度"的指标——数值越低说明文本越" predictable"、越像正常语言。早期系统（2019 年的 CCNet）用困惑度来做质量筛选：在一个高质语料（如 Wikipedia）上训练一个小型语言模型，然后用它给每个文档打分。困惑度低的文档保留——因为它们更像 Wikipedia 的风格，质量通常更高。

## 9. Token 是什么？

Token 是模型处理文本的基本单位。英文大约 0.75 个词 = 1 个 token，中文大约 0.5-1 个汉字 = 1 个 token。"Hello world" = 2 个 token。"你好世界" = 4 个 token。模型的参数量和训练数据量通常以 token 为单位衡量。

## 10. 数据-参数比

模型参数量与训练 token 量的比值。Chinchilla 法则认为最优比是 **1:20**（1 个参数对应 20 个训练 token），但后来的实践证明这个比例可以更大。MiniCPM 做到了 1:160——小模型用海量数据训练来弥补参数不足。比值越高，说明数据利用越充分。

### FineWeb

HuggingFace 2024 年 6 月发布的 15T token 高质量预训练数据集。不只是数据集——更重要的是一套完全透明的数据工程方法论和消融实验系统（ablation），让行业第一次能精确量化"数据处理的每一步提升了多少模型性能"。它的子集 FineWeb-Edu（1.3T token）专门筛选教育质量高的内容，在推理基准上显著超越更大规模的数据集。

### Model Collapse（模型崩溃）

合成数据训练过多后模型性能开始回退的现象——表现为输出多样性下降、长尾知识丢失。ICML 2025 多篇论文证实：当合成数据占比超过 30-40% 后，边际收益递减甚至负增长。根本原因是模型生成的数据受限于自身表达能力和偏差。这是合成数据路线的核心风险。

## 参考来源

- **Training Compute-Optimal Large Language Models (Chinchilla)** (Hoffmann et al., DeepMind, 2022) — 提出 Chinchilla 缩放法则（最优参数-数据比为 1:20），终结了"只堆参数不堆数据"的时代 — https://arxiv.org/abs/2203.15556
- **Documenting Large Webtext Corpora: A Case Study on the Colossal Clean Crawled Corpus (C4)** (Dodge et al., 2021) — 首次系统审计 C4 数据集，揭示大规模网络语料的组成偏倚和过滤策略影响 — https://arxiv.org/abs/2104.08758
- **CCNet: Extracting High Quality Monolingual Datasets from Web Crawl Data** (Wenzek et al., Facebook AI, 2019) — 用困惑度做质量过滤的开创性方案，训练小型语言模型给 CommonCrawl 文档打分 — https://arxiv.org/abs/1911.00359
- **DCLM: DataComp for Language Models** (Li et al., 2024) — 用模型本身筛选数据的标准化基准，2.6T 精选 token 效果媲美 15T 普通 token — https://arxiv.org/abs/2406.11794
- **The FineWeb Datasets: Decanting the Web for the Finest Text Data at Scale** (Penedo et al., HuggingFace, 2024) — 15T token 高质量预训练数据集 + 消融实验，首次精确量化每一步清洗的性能贡献 — https://arxiv.org/abs/2406.17557
- **Dolma: an Open Corpus of Three Trillion Tokens for Language Model Pretraining Research** (Soldaini et al., AI2, 2024) — 3T token 开放预训练语料，覆盖从原始网页到最终配比的全链路可复现 — https://arxiv.org/abs/2402.00159
