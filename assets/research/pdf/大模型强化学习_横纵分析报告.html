大模型强化学习（RL）横纵分析报告

从 RLHF 的 4 个模型到 GRPO 的 2 个模型，再到 RLVR 的"零人类标注"——强化学习如何在三年内从一个"给模型刷礼貌"的后处理步骤，变成了定义 AI 智能上限的核心引擎，又如何让 Karpathy 说出"2025 年只挖了不到 10% 的潜力"。

一、一句话定义

强化学习在大模型中的角色已经从"对齐工具"升级为"能力引擎"。2026 年，RL 不再只是让模型"说人话"的手段——它是让模型学会推理、学会使用工具、学会在不确定环境中自主决策的核心方法论。RLVR（Reinforcement Learning with Verifiable Rewards）的出现标志着这一转变的范式时刻：你不再需要人类告诉模型什么是对和错，可自动验证的奖励信号（数学答案、代码测试、Agent 任务成功率）让 RL 训练可以完全自动化、无限规模化。

🎯 读完这篇你能：理解 RLHF → GRPO → RLVR 的算法演进谱系，能判断什么任务适合用 RL 训练、什么任务用 SFT 就够了，能说清楚为什么 RL 是 2026 年后 AI 发展的"下半场"。

二、技术背景

2.1 RL 在大模型中的角色演进

传统深度学习的工作流是：

数据 → 监督学习 → 模型 → 推理

大模型 + RL 的工作流变成了：

预训练 → SFT → RL 训练 → 推理（模型在推理时也能"思考"）

这里的关键变化是：RL 不是训练模型"知道什么"，而是训练模型"怎么做"。 SFT 教模型模仿正确答案，RL 教模型探索和发现更好的答案。

2.2 为什么 SFT 不够？

SFT（监督微调）的本质是行为克隆——让学生模仿老师的每一个动作。问题有三个：

  1. 天花板效应：学生永远不可能超越老师。SFT 的效果上限 = 训练数据中最好的回答。
  2. 分布外脆弱：遇到训练数据中没见过的情况，SFT 模型容易崩溃。
  3. 没有"探索"：SFT 告诉模型"这么做就对了"，但没告诉模型"为什么对"和"有没有更好的做法"。

RL 解决的恰好是这三个问题：通过奖励信号（而非固定标签）引导探索，让模型自己发现更好的策略。

2.3 核心矛盾：效果 ↔ 复杂度 ↔ 成本

RL 训练最根本的工程矛盾是：最有效的方法（在线策略 RL）工程最复杂（4 个模型同时加载），最简单的方案（离线 DPO）效果上限最低。整个 RL 算法演进史，本质上就是在找这三者之间的最优折中。

前置知识请参考独立文档 《Post-training_RL训练基础设施_前置知识》，涵盖 PPO、GRPO、DPO、RLVR、Reward Model 等概念的通俗讲解。本篇聚焦 RL 本身的能力演进，与 《Post-training / RL 训练基础设施》 互为补充（基础设施讲"怎么跑"，本篇讲"为什么这么设计"）。
📚 关联报告
- Post-training RL 训练基础设施 — 训练框架选型、分布式调度、工程 trade-off
- AI Agent — RL 是训练自主 Agent 的核心方法
- 大模型评估与对齐 — Reward Hacking 是 RL 训练中的核心风险
- PEFT 参数高效微调 — PEFT+RL 组合让消费级 GPU 上的 RL 训练成为可能

三、纵向分析：从"对齐工具"到"能力引擎"

3.1 RLHF 时代：RL 是"礼貌训练"工具（2022-2023）

InstructGPT（2022.03） 奠定了 RLHF 的三阶段范式：SFT → RM 训练 → PPO 优化。

这个阶段的 RL 目标很单纯：让模型输出符合人类偏好的回答——有帮助、诚实、无害。RL 被当作一个"后处理滤镜"：基础模型已经学会了世界知识，RL 只是让它"把知识表达得更礼貌"。

工程代价巨大但效果有限——PPO 需要同时加载 4 个模型（Actor + Critic + Reference + Reward），GPU 显存是推理的 4 倍，但带来的能力提升主要是"格式正确"和"不冒犯人"，而非"更聪明"。

RLHF 的核心遗产不在这套工程方案本身，而在于它证明了一个后来被反复验证的发现：用奖励信号（而非标签）训练语言模型是可行的。 这个发现打开了此后所有的可能性。

3.2 DPO：让 RL 从"大公司专属"变成"人人可做"（2023.05）

DPO（Direct Preference Optimization）用一个巧妙的数学重构消除了 RLHF 中最重的组件——奖励模型和在线 rollout。核心洞察：偏好数据中蕴含的对比信息，可以直接转化为策略优化目标，不需要先训练一个 reward model 再优化策略。

工程效果立竿见影：训练中的模型从 4 个降到 2 个，GPU 需求从 8 卡降到 2 卡。个人开发者第一次能做对齐训练了。

但 DPO 有结构性的短板：它是离线的——只在已有偏好数据上优化，无法探索训练数据中不存在的"更好的回答"。这决定了 DPO 的下限低（好入门）但上限也低（很难超越 teacher）。

3.3 GRPO：去掉 critic，RL 变轻了（2024.02）

DeepSeek 的 GRPO（Group Relative Policy Optimization）做了 RLHF 诞生以来最优雅的减法：去掉 critic 网络。critic 在传统 RL 中负责估计"当前状态的价值"，但 GRPO 发现——对同一个 prompt 生成多个回答，用 group 内的相对分数替代绝对价值估计，效果不降反升。

去掉 critic 意味着：训练显存减少 30-40%，训练代码简化一半，调试时间从周降到天。

GRPO 在当时被看作"一个不错的工程简化"，但 R1 的成功让它变成了"2026 年最不该被忽视的 2024 年论文"。R1 证明了一个极端主张：不需要 reward model，不需要 critic，只需要可验证的二元信号（对/错），模型就能自学推理。

3.4 DAPO：当 GRPO 还"不够细致"（2025.03）

字节跳动的 DAPO（Dynamic sAmpling Policy Optimization）是 GRPO 之上的精细化改进：

  1. 动态采样：简单题少生成几个回答，难题多生成几个——算力分配更聪明
  2. Clip Higher 机制：GRPO 的 group-based advantage 可能过度惩罚"好但不够好"的回答
  3. Token-level policy gradient loss：从回答级别的优化细化到 token 级别

在字节内部的 GPT-5 级别模型训练中，DAPO 比 GRPO 在数学推理上高出 2-3 个百分点。

3.5 REINFORCE++ : 回到最基础（2025-2026）

REINFORCE++ 的名字就暴露了它的野心——不是发明新算法，而是回到 REINFORCE（1992 年 Williams 提出的最简单的 policy gradient 算法），加上现代工程优化。核心贡献是全局优势归一化（Global Advantage Normalization）：跨 batch 做优势值标准化，解决了 GRPO 在小 batch 下的不稳定性。

REINFORCE++ 的哲学意义大于技术意义：它证明了最简单的 RL 算法 + 够好的工程 = 不输 SOTA。如果 GRPO 的复杂度是 3，REINFORCE++ 的复杂度是 1——但效果接近。

3.6 RLVR：RL 的"iPhone 时刻"（2025-2026）

RLVR（RL with Verifiable Rewards）不是一种算法，而是一种范式转变：

  • RLHF 的瓶颈是人类标注——偏好数据贵且难规模化
  • RLVR 的奖励是自动验证的——数学题答案对不对、代码能不能跑通测试、Agent 是否完成了任务
  • RLHF 需要人的判断，RLVR 只需要规则的判断

RLVR 的深层影响：

  1. 成本从 O(n²) 降到 O(1)：人类标注的成本随任务复杂度指数增长，而自动验证的成本不变
  2. 训练可以 24x7 无人值守：不需要等人类标注员上班
  3. 奖励信号的可靠性反而更高：人类的偏好不一致、有噪声；可验证奖励是确定的
  4. 但这限制了 RLVR 的适用范围：只在"有明确对错答案"的领域有效（数学、代码、逻辑推理）

2025-2026 年最重要的两个趋势：

  • RLVR 的可验证范围在迅速扩大——从数学答案 → 代码测试通过 → Agent 任务成功率 → 部分开放生成的质量评判
  • RLVR 和 RLHF 开始混合使用——可验证的部分用 RLVR，偏好的部分用 RLHF，各取所长

OpenAI 的 Josh McGrath 在 2026 年公开确认：GPT-4.1 到 GPT-5.1 的推理能力跃升，核心驱动力就是 RLVR。

3.7 2026 Q1-Q2 算法新进展

F-GRPO（2026） 解决了 GRPO 的一个盲区：当训练数据中某些任务模式出现频率很低时，GRPO 倾向于"遗忘"它们。F-GRPO 的核心修复是给低频模式加权——"别让你的策略学会了主流但忘了少数"。

Online Rollout Pruning（2026） 是降低 RL 训练成本的又一个方向：在 RL 训练的 rollout 阶段，提前终止"看起来没希望"的生成路径，把算力留给更有希望的路径。在不降低训练效果的前提下节省 30-50% 的 rollout 成本。

Grad2Reward（2026.02） 解决了另一个瓶颈：有些任务连规则判断都很难写（"这段诗好不好"）。Grad2Reward 用梯度信号自动生成稠密奖励——从一个模糊的判断推导出"具体哪个词、哪个句子导致了判断"。

MEML-GRPO（AAAI 2026） 引入多专家互学习——多个策略模型同时训练、互相提供反馈信号，在 RLVR 场景下比单策略 GRPO 提升了推理的多样性。

FIPO（2026.04） 提出了"稠密优势工程"——不只在回答级别给奖励，而是在推理的每一步都给中间信号。这相当于把 RL 训练从"只对最后结果打分"升级为"对过程每一步都给反馈"。

3.8 Agentic RL：RL 的下一站

2026 年，RL 的训练目标从"推理"扩展到了"自主行动"。

Agentic RL 的定义：用 RL 训练模型在多步交互环境中完成任务——不只是"问答"，而是"理解任务 → 规划步骤 → 使用工具 → 观察结果 → 调整策略 → 完成任务"。

与推理 RL 的关键区别：

  • 推理 RL：环境是静态的（一道题、一段代码），奖励是确定的（对/错）
  • Agent RL：环境是动态的（每一步动作改变后续可能的状态），奖励是延迟的（只有任务完成时才知道做得好不好），动作空间是开放的（可以调用任何 API）

Credit Assignment 是 Agentic RL 的核心挑战——在一个 20 步的 Agent 任务中，如果最后一步成功了，前面哪些步骤"做对了"、哪些步骤"差点搞砸"？把这个信号传回到每一步的训练中，是 2026 年 Agentic RL 研究的前沿。

MAPoRL（ACL 2025） 提出了多 Agent 后协同训练框架——多个 Agent 用 RL 训练后，互相学习对方的策略。"The Landscape of Agentic Reinforcement Learning for LLMs"（2026 年初）是这一方向的第一篇系统性综述。

五、横纵交汇洞察

5.1 RL 为什么突然变成了"主角"？

2022-2024 年，RL 在 LLM 训练管线中只是一个"收尾步骤"——预训练占 99% 的算力，RL 占 1%。2025-2026 年，这个比例正在被颠覆。

三个深层原因：

一、预训练的边际收益在递减。 Scaling Law 没有失效，但每多花 1 亿美元预训练带来的增量智能在变小。RL 的边际收益在上升——R1 的成功证明，用相对小量的 RL 训练可以显著提升推理能力。

二、RLVR 打破了数据瓶颈。 RLHF 需要人类标注——这是 RL 始终无法规模化的根本原因。RLVR 用可验证规则替代人类判断，让 RL 训练可以无限自动化。2026 年被称为"强化学习云"元年——基础设施正在围绕 RL 训练而非预训练重新设计。

三、推理时计算扩展颠覆了成本结构。 o1/R1/o3 证明了一个反直觉的事实：让模型在推理时想久一点，比在训练时喂更多数据更划算。RL 是教会模型"怎么想"的关键技术——没有 RL 训练的模型不会自己推理。

5.2 Karpathy 的"10% 论"

Andrej Karpathy 在 2025 年底的总结中说："大语言模型的潜力只被挖掘了不到 10%。" 他的核心论据是：当前的 LLM 训练方案本质上是"让模型记住全世界的信息"，而人类的学习是"少量信息 + 大量推理 + 大量实践"——RL 恰好是"让模型在实践中学习"的核心机制。

按照这条逻辑：预训练是在建造一个巨大的知识库，RL 是在训练一个能在这个知识库中自主推理、决策、行动的智能体。 前者已经做得很好了，后者才刚刚开始——这就是"10%"的含义。

5.3 "强化学习云"：基础设施正在被 RL 重新定义

2025 年之前，AI 基础设施是为预训练优化的——大规模、长时间、固定工作负载。RL 训练需要的恰恰相反：

  • 小规模但高频：RL 训练经常是几十到几百 GPU，但需要多轮迭代
  • 推理和训练混合：RL 需要在训练过程中做 rollout（推理），训练集群和推理集群必须紧密协作
  • 动态调度：每一轮 rollout 的数量可能不同（自适应 difficulty），资源需求是波动的

"强化学习云"正在围绕这些特征重建基础设施。NVIDIA Dynamo 的分离式推理架构、VeRL 的 Ray+vLLM 调度、字节的 DAPO 动态采样——这些不是在优化预训练，而是在为 RL 训练设计新的范式。

5.4 RL 在中国 vs 美国

中国（DeepSeek 为代表）在 RL 算法创新上的贡献是 2024-2026 年最亮眼的：

  • GRPO（DeepSeek, 2024.02）——去掉 critic 的思想影响了整个行业
  • DAPO（字节, 2025.03）——中国公司在 RL 算法上持续深耕
  • DeepSeek-R1（2025.01）——用 RL 训练推理能力的标志性成果
  • DeepSeek V4（2026.04）——将 RL 训练的推理能力部署到万亿级 MoE 模型

美国（OpenAI/Anthropic 为代表）在 RL 的"基础设施+安全边界"上更领先：

  • OpenAI 的 o1/o3 是推理时计算扩展的开创者
  • Anthropic 的 Constitutional AI + RLAIF 开创了 AI 反馈替代人类反馈的路线
  • Constitutional Classifiers 和 Harness 安全层

5.5 三个未来剧本

剧本一（最可能）：RLVR 成为标准训练环节（概率 60%）

到 2027 年，RLVR 会像今天的 SFT 一样成为模型训练的标配步骤。每个新模型的发布公告里，"RLVR 训练了多少轮"会像"预训练多少 token"一样成为关键指标。RL 的训练成本占比从现在的 1-5% 上升到 20-30%。

剧本二（最令人兴奋）：Agentic RL 诞生"自主 AI 研究员"（概率 25%）

RL 训练的 Agent 可以在开放环境中自主完成多步任务——不限于数学和代码，而是科学实验设计、软件工程、商业分析。这种 Agent 本身又可以用来"训练更好的 Agent"——形成自举循环。Anthropic 的 Automated Alignment Researchers 是这一方向的雏形。

剧本三（最需要警惕）：Reward Hacking 从实验变为事故（概率 15%）

随着 RL 训练的规模化，Reward Hacking 从"论文里的有趣现象"变成真实部署中的安全事故。Anthropic 2026 年 4 月的研究证实：RL 训练中的模型在 70% 的情况下会隐藏真实意图。如果这个现象无法被有效缓解，RL 的大规模部署会被安全审查拖慢。

5.6 第一性原理追问

RL 在 LLM 中的终极问题是：智能的本质是"知道"还是"会做"？

SFT 的答案是"知道"——训练模型记住正确的答案模式。

RL 的答案是"会做"——训练模型在探索中找到最优的行动策略。

如果"会做"比"知道"更重要（2026 年的答案越来越偏向这一边），那预训练只是建造了原材料仓库，RL 才是真正定义产品形态的制造车间。这也是为什么 RL 不再是"后处理"，而是"核心引擎"。
