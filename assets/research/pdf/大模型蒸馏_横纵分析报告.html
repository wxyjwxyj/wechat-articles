大模型蒸馏 / Knowledge Distillation 横纵分析报告

从 Hinton 2015 年的"暗知识"到 DeepSeek V4 的 On-Policy Distillation（OPD）——一卷蒸馏如何从"压缩模型的小技巧"变成了"制造智能的核心工艺"。

一、一句话定义

知识蒸馏（Knowledge Distillation）是让一个"学生"（student）模型学习"老师"（teacher）模型的输出，以更小的参数量逼近老师能力的方法集合。它在 LLM 时代的角色已经远超"事后压缩"——2026 年，蒸馏正在成为 post-training 的核心环节：不是把训练好的大模型"缩小"，而是用大模型来制造更好的小模型。

🎯 读完这篇你能：理解从 Hinton 的 soft target 到 DeepSeek V4 OPD 的四代蒸馏演进，能判断什么时候用黑盒蒸馏、什么时候用白盒蒸馏、什么时候该走 On-Policy。

二、技术背景

2.1 蒸馏的直觉

想象一个人类专家教学生。专家的价值不只是"给出正确答案"——而是"展示完整的思考过程"。学生在学习过程中不仅能学会解这道题，还能学会专家的解题思路。蒸馏的核心洞察完全相同：老师输出的"如何得到答案"比"答案本身"更有教学价值。

三、纵向分析：从"软标签"到"在线蒸馏"

3.1 第一代：Soft Target 蒸馏（2015）

Hinton 在 "Distilling the Knowledge in a Neural Network"（NIPS 2014 Workshop）中的核心洞察：老师输出的概率分布比硬标签包含更多信息。

比如一张猫的图片——硬标签是 [猫: 1, 狗: 0, 兔子: 0]。但老师输出的 soft target 是 [猫: 0.8, 狗: 0.15, 兔子: 0.05]——这个分布告诉学生："猫"和"狗"比"猫"和"兔子"更相似。这种"暗知识"（Dark Knowledge）是 teacher 从海量数据中学到的泛化能力。

3.2 第二代：数据蒸馏 = 合成数据训练（2023-2024）

LLM 时代的第一波蒸馏革命不是改进 Hinton 的 logit-level 方法，而是走了一条更简单直接的路：让 teacher 生成数据，用这些数据做 SFT 训练 student。

原因很现实：闭源 API（GPT-4、Claude）不可能开放 logits，你只能拿到 text output。那既然拿不到"暗知识"，能不能让 teacher 生成的 text 本身质量足够高、足够教学化？

Phi 系列（Microsoft, 2023-2024） 就是这个路线的标志：用 GPT-4 生成"教科书质量"的合成数据，挑出最好的数据训练小模型。Phi-1 用 1.3B 参数在编程任务上超越了 12B 的模型。

DeepSeek R1-Distill（2025.01） 是数据蒸馏的巅峰：用 R1 671B 生成包含完整推理链的数据，训练 Qwen 32B/7B 和 Llama 8B。关键发现：推理能力——不是知识——可以通过蒸馏传递。 学生不需要自己探索，只需要"看"老师怎么思考。

3.3 第三代：黑盒在线蒸馏（2025）

数据蒸馏的问题是离线——学生学的是老师在训练时生成的固定数据。但老师的输出质量在生成时是固定的，学生无法在训练过程中"问老师新问题"。

Black-Box On-Policy Distillation（GAD, 2025.11） 是微软提出的方案：学生自己生成回答 → 送给老师（作为黑盒 API）评分 → 用评分信号更新参数 → 学生再生成、再问老师。这恢复了 RLHF 的"在线探索"能力但不要求老师是白盒。

SODA（Semi On-Policy Distillation, 2026.04） 进一步优化了效率：不是每轮都问老师，而是混合使用离线数据和在线数据，减少昂贵的 API 调用。

3.4 第四代：DeepSeek V4 OPD — 蒸馏成为核心工艺（2026.04）

DeepSeek V4 的技术报告披露了一个颠覆性的 post-training 架构转变：放弃混合 RL，改用 On-Policy Distillation（OPD）。

传统 RL 的瓶颈：需要海量训练数据、训练过程不稳定、reward hacking 风险。

V4 OPD 的做法：

  1. 训练多个"专家模型"（每个在特定领域做到极致，如数学、代码、安全）
  2. 学生模型在训练中自主生成回答
  3. 将学生的回答送给对应的专家模型评分
  4. 蒸馏多个专家的能力到一个统一的学生模型中

效果：多专家知识融合到一个模型中，推理能力不再需要繁琐的 RL 训练。

"Rethinking On-Policy Distillation"（清华 NLP, 2026.04）为 OPD 提供了系统的理论分析和工程指南，识别了三类常见失败模式并给出了修复方案。

3.5 Self-Distillation：不需要外部老师

Self-Distilled Reasoner（2026.01） 提出了一个反直觉的方向：不需要外部 teacher。学生通过反复"练习→反思→改进"的循环，自己蒸馏自己——用自己的高质量版本来训练自己的下一版。

PAINT（2026.04）改进了 self-distillation 的效率——用部分完成（而非完整）的解来训练。这降低了"自己蒸馏自己"的数据准备成本，被证明在推理任务上不输传统蒸馏。

Self-Distilled RLVR（2026.04） 是另一个创新——将 RLVR 的训练信号（可验证奖励）同时用作蒸馏的监督信号。不需要人类标注，不需要外部 teacher，只需要可验证的奖励规则。

五、横纵交汇洞察

5.1 蒸馏 = 知识搬运，不是知识创造

蒸馏不能让学生变得比老师更强——至少理论上是这样。但实践中，学生经常在特定任务上超过老师。原因在于：蒸馏的筛选过程本身就是一种"质量提升"——老师生成的错误答案不会被蒸馏到学生中，而老师教不了学生的东西恰好是老师的弱势。

5.2 On-Policy Distillation 是新范式

DeepSeek V4 的 OPD 是 2026 年最重要的蒸馏范式转变：不再把蒸馏当作"训练后的附加步骤"，而是作为 post-training 的核心管线。

OPD 和 RL 的关系正在变得微妙：如果 OPD 能做到接近 RL 的探索能力，同时比 RL 更稳定、更可控，那 RL 在 LLM post-training 中的角色会被重新定义。

5.3 蒸馏的经济学

  • 训练一个 671B 的 teacher：数千万到数亿美元
  • 蒸馏到 7B-32B 的 student：数万美元（API 调用成本 + GPU 时间）
  • Student 的推理成本：teacher 的 1/100 到 1/1000
  • 在特定任务上，student 能达到 teacher 80-95% 的能力

投资回报率极高。蒸馏的成本在 teacher 训练完成后几乎可以忽略不计，但带来的推理成本节约是永久性的。
