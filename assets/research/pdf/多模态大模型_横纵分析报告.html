多模态大模型横纵分析报告

从 CLIP 到 GPT-4o，从"看图说话"到"全模态统一"，大模型如何长出了眼睛和耳朵。

一、一句话定义

多模态大模型（Multi-modal Large Language Model, MLLM）是在传统文本大模型基础上扩展了图像、视频、音频等模态理解与生成能力的 AI 模型。它的核心挑战是：如何让一个以文本为中心训练的神经网络，"看懂"像素构成的世界，并在不同信息形态之间建立深层的语义关联。

🎯 读完这篇你能：理解视觉编码器-模态连接器-LLM 三件套的工作原理，能评估一个多模态系统的能力边界和架构取舍。

二、技术背景

核心矛盾：文本和图像是两种完全不同的"语言"

文本是离散符号——每个字是一个 token，有明确的边界和语法规则。图像是连续信号——像素构成的矩阵，没有天然的"词边界"，语义信息隐藏在颜色、纹理、形状的组合中。

让一个模型同时理解这两种"语言"，需要解决三个关键问题：

  1. 视觉编码：怎么把图片变成模型能处理的 token 序列？
  2. 模态对齐：怎么让"狗的图片"和"狗的文字"在模型内部指向同一个语义概念？
  3. 融合推理：怎么让模型同时利用视觉信息和文本信息做综合推理？

这三条线分别对应多模态模型的三大组件：

组件功能典型方案
1视觉编码器图片 → token 序列ViT → SigLIP → InternViT
2模态连接器视觉 token → LLM 可理解MLP → Q-Former → Perceiver Resampler
3LLM 本体融合推理文本 LLM + 视觉 token 输入

📚 关联报告

- AI 视频生成 — 视频生成是多模态模型从静态到动态的自然扩展
- LLM 基础大模型 — 多模态模型以文本 LLM 为核心，通过视觉编码器扩展感知能力
- Transformer — ViT 将 Transformer 泛化到视觉，是多模态模型的架构基础

三、纵向分析：从 CLIP 到全模态统一

3.1 奠基期：视觉-语言表征学习（2021-2022）

多模态大模型的起点，不是"给 LLM 加眼睛"，而是先解决一个更基础的问题：怎么让计算机学会"看图说话"？

2021 年 2 月，CLIP（OpenAI） 给出了一个极简但强大的答案。Alec Radford 和 Ilya Sutskever 的团队用 4 亿对互联网图文数据训练了一个对比学习模型，核心任务极其简单：给定一张图和一段文字，判断它们是否匹配。

CLIP 的贡献在于两件事：一是证明了弱监督的图文对数据足以学到可迁移的视觉表征——不需要人工标注，互联网上天然存在海量的图文对（新闻配图、产品描述等）。二是它的零样本分类能力——不需要任何训练数据，直接给 CLIP 看一张猫的图片，它就能在 ImageNet 的 1000 个类别中找到"猫"。

回头看，CLIP 的意义超越了"又一篇好论文"。它为整个多模态领域提供了一种"共同语言"——后来的几乎所有多模态模型，不管架构多复杂，都以 CLIP 或其变体作为视觉编码的基础。

2022 年 4 月，Flamingo（DeepMind） 迈出了从"视觉理解"到"多模态对话"的关键一步。Jean-Baptiste Alayrac 等人的核心创新是桥接预训练的纯视觉和纯语言模型，通过门控交叉注意力（Gated Cross-Attention）在 LLM 的每一层注入视觉信息。

Flamingo 的架构设计后来被证明非常有远见：它处理任意交错的图文序列，支持图像和视频输入，而且 LLM 本身不需要微调——视觉信息通过交叉注意力"旁路"注入。这种设计让 Flamingo 不需要大规模多模态预训练，就可以在少量示例下（in-context few-shot）理解图片内容。

Flamingo 的局限也很明显：架构过于复杂（每层交叉注意力），推理成本高，而且门控机制本身引入了额外参数。但它证明了"把已经存在的单模态模型拼在一起"是一条可行的技术路线。

这个阶段的本质是"积攒积木"：CLIP 贡献了视觉编码器，Flamingo 贡献了跨模态连接的架构思路。2023 年之前，多模态模型还处于"从零开始造轮子"的阶段。

3.2 爆发期：高效连接与指令微调（2023）

2023 年是多模态大模型的元年。不是因为某篇论文突然突破，而是几条独立的技术线在这一年同时成熟，产生了化学反应。

2023 年 1 月，BLIP-2（Salesforce） 的 Q-Former 解决了"连接器效率"问题。Q-Former 的核心设计是一个"信息瓶颈"——用一组可学习的 query 向量从视觉编码器的输出中提取关键信息，压缩成少量 token 后送入 LLM。

Q-Former 的巧妙之处在于"模块化"：视觉编码器冻结、LLM 冻结，只有 Q-Former 本身被训练。训练分三个阶段，前两个阶段在图文数据上预训练 Q-Former，第三阶段才把它接到 LLM 上。只用少量可训练参数就超越了 Flamingo-80B 在 VQAv2 上 8.7% 的性能——可训练参数只有 Flamingo 的 1/54。

2023 年 4 月问世的 LLaVA（UW / Microsoft） 则走向另一个极端：最大程度的简化。Haotian Liu 和 Chunyuan Li 用最简单的 MLP 做连接器，直接把 ViT 的输出 token 映射到 LLM 的嵌入空间。结构简单到让人怀疑"这也行？"——但配上 GPT-4 生成的高质量指令数据，LLaVA 证明了简单的架构+好的数据可以达到惊人的效果。

但 LLaVA 和 Q-Former 走的是同一条路的不同分支：都是用轻量级连接器把视觉 token 映射到 LLM 空间。这条路后来被证明是多模态模型最实用的路线——简单、高效、可扩展。

2023 年 3 月，Google 的 SigLIP 提出了一个看似微小但影响深远的改进：用 sigmoid loss 替代 CLIP 的 softmax 对比损失。CLIP 需要在一个 batch 内做全局归一化（计算所有图文对的相似度），batch size 跟计算量平方相关，导致小 batch 训练效果差。SigLIP 的 sigmoid loss 逐对计算，不依赖 batch 内其他样本——由此，可以用小得多的 batch 达到同等效果。

与此同时，InstructBLIP（Salesforce, 2023.05） 在 BLIP-2 基础上引入了"指令感知"的 Q-Former——根据用户的指令提取针对性特征，而不是一股脑把视觉信息都传给 LLM。这个直觉很简单：问"这里有什么动物？"和问"这个场景在室内还是室外？"需要从图片中提取完全不同的信息。

8 月，阿里巴巴发布了 Qwen-VL——一个三阶段训练的视觉语言模型（预训练 → 多任务微调 → 对话微调），支持视觉定位（说"狗" → 指出狗在图中的位置）和文本阅读。Qwen-VL 的意义在于它是国内第一个系统级的多模态工作，后续 Qwen-VL 系列的一路领先从这时就已埋下伏笔。

CogVLM（清华/智谱, 2023.11） 走了一条不同的路：不是"浅层连接"（MLP 或 Q-Former 都在 LLM 外面接），而是在 LLM 的 Attention 和 FFN 层中引入可训练的"视觉专家"模块。这种"深度融合"的设计在理论上更优，但代价是实现复杂、训练成本高。

这个阶段的本质是"寻找最简路径"：从 Flamingo 的复杂桥接到 LLaVA 的简单拼接，行业在用最快的速度收敛到"够用且简单"的方案。Q-Former 和 MLP 两条路线并行发展，但方向一致——轻量连接器 + 冻结视觉/LM。

3.3 商用化爆发（2023 年末–2024）

2023 年 9 月 25 日，GPT-4V——行业等待已久的时刻。OpenAI 发布了第一个商用多模态模型，让 GPT-4 能够"看"图片。

GPT-4V 的发布是一个分水岭。在此之前，多模态大模型是学术论文里的概念。在此之后，它是每个开发者都能调用的 API 能力。开发者社区迅速发现了它的能力边界：强大的通用图像理解、不错的 OCR、在医学影像和专业图表上的出色表现——但也存在幻觉、对细微视觉差异不敏感等问题。

3 个月后，Google DeepMind 发布 Gemini 1.0（2023.12），技术报告标题直接写着"原生多模态"。Gemini 从设计之初就不是"文本模型+视觉能力"的组合，而是所有模态一起训练的。这在当时是一个冒险的技术选择——因为它需要从头开始训练，成本极高，且无法利用已有的纯文本 LLM。

但原生多模态的优势在后续迭代中逐渐显现：Gemini 1.5 Pro（2024.02）支持 1M token 上下文，Gemini 2.0 强调多模态输出和 Agent 能力。Google 的算力储备（TPU 集群）让这条昂贵的路线变得可行。

2024 年 3 月，Claude 3 Vision 姗姗来迟。Anthropic 明确走"稳妥路线"——先在文本模型上做到极致，再把视觉能力接上去。Claude 3 的视觉能力在发布时被广泛认为是最精确的，尤其在图表和文档理解上。但它的架构选择（后期拼接）也决定了它的上限——在需要深度视觉-语言融合的任务上不如原生多模态方案。

5 月，GPT-4o 发布，OpenAI CEO Sam Altman 在 X 上只发了一个词："her"。"o"代表"omni"——全模态。GPT-4o 统一了文本、图像、音频的处理，所有输入输出由同一个神经网络处理。音频响应延迟最低 232ms，接近人类对话反应时间。

GPT-4o 的发布标志着一个判断：多模态不是"附加功能"，而是模型的核心能力。单独的文本模型、视觉模型、语音模型正在被统一的"全模态模型"取代。

2024 年 4 月，InternVL 1.5（上海 AI Lab） 发表时，论文标题就很直白："我们还离 GPT-4V 有多远？"答案是：越来越近了。InternVL 1.5 的三大改进——更强的视觉编码器（InternViT-6B 持续学习）、动态高分辨率（支持 4K 分辨率输入）、高质量双语数据集——让它在 18 个基准中的 8 个上达到 SOTA。SOTA 不意味着跟 GPT-4V 一样好，但差距从"不可逾越"缩小到了"可以追赶"。

9 月是开源多模态的双响炮。Qwen2-VL（阿里）引入了"朴素动态分辨率"——不缩放、不裁剪，按原生宽高比处理图片。Llama 3.2 Vision（Meta）首次将视觉能力引入 Llama 系列。前者在做技术突破，后者在做生态扩展。

动态分辨率是 2024 年最重要的技术演进之一。之前的多模态模型把所有图片缩放到固定尺寸（如 336x336），大尺寸的文档、高分辨率的图片中的细节全部丢失。Qwen2-VL 的方案：根据图片的实际尺寸动态决定切分方式，每个子图块（tile）独立编码再拼接。这看起来是实现细节，实际上彻底改变了模型对高分辨率场景（文档分析、医学影像、专业图表）的处理能力。

这个阶段的本质是"商业验证"：GPT-4V 和 GPT-4o 验证了多模态的商业可行性，Gemini 验证了原生多模态的技术可行性，Qwen2-VL 和 InternVL 验证了开源可以追赶闭源。

3.4 成熟与普及（2025-2026）

2025-2026 年，多模态大模型完成从"前沿技术"到"基础能力"的转变——但这个完成的过程伴随着一个令人意外的技术回潮。

技术层面的收敛与回潮：2023 年的"连接器之争"基本平息，主流偏向动态分辨率 + ViT/SigLIP 变体 + MLP + LLM。但 2025 年出现了一股"原生多模态回归"的浪潮——从 Apple、Meta、商汤到智源，多个团队同时开始挑战"CLIP+连接器+LLM"的分离式范式。

Apple 的 Scaling Law 研究（2025.04） 是最尖锐的挑战者。Apple 团队发现：早融合（early fusion）+ MoE 架构在大规模训练下的性能上限高于传统的后融合方案。这预示着当前的分离式架构（CLIP → 连接器 → LLM）可能在规模扩展时到达性能天花板——"视觉编码器本身就是瓶颈"。

Meta Llama 4（2025.04） 直接在旗舰模型上投了原生多模态的票——MoE 架构 + native multimodal，支持 10M token 上下文，在多项基准对标 GPT-4o。这不是一篇实验论文，而是 Meta 最核心产品模型的路线选择。

国内原生多模态三箭齐发：商汤 NEO（2025）号称"世界首个完全抛弃视觉编码器的原生多模态架构"——视觉和语言在 token 空间完全统一；智源 Emu3.5（2025）用自回归技术统一理解与生成；Tuna-2（Meta, 2026.04）更进一步提出"像素嵌入可直接取代视觉编码器"。

这些工作在 2026 年还处于追赶阶段——性能最强的仍然是 Qwen3-VL 等后期拼接模型——但它们代表了一股正在积聚的力量：分离式架构的"后期拼接"可能不是多模态的终局。当视觉编码器成为性能瓶颈时，直接从像素出发的原生多模态架构会越来越有吸引力。

技术进步的另一方面：视觉 token 压缩（InternVL-X, 2025.03）、单体式架构尝试（Mono-InternVL）、渐进式训练策略的成熟。SigLIP 2（Google, 2025.02）引入 NaFlex 架构——支持动态分辨率和可变 patch 大小的视觉编码，让同一编码器处理不同尺寸的视觉输入，已被 PaliGemma 2 等新 VLM 采用。

产品层面的渗透与扩展：多模态能力不再是"高端模型"专属——Claude Haiku 也支持图像输入，GPT-4o-mini 同样。Gemini 2.5 Flash 低价视觉 API 把每张图片的分析成本打到接近零。

但 2025 年最被低估的多模态事件不在"理解"端——它在"生成"端。GPT-4o 在 2025 年 3 月上线了原生图像生成能力，不再依赖 DALL-E，实现了"一个模型理解也能生成图像"。首周 1.3 亿用户生成了 7 亿张图片。Gemini 2.0 Flash 同期支持了原生图像输出。Qwen3.5-Omni（2026.04）更进一步——文本、图片、音频、视频四种模态统一输入输出。全模态不再是愿景，而是产品现实。

多模态 Agent 的登场：2025 年是多模态从"看一看"到"动一动"的拐点。Anthropic Computer Use 让 Claude 看屏幕、操作鼠标键盘，微软 OmniParser V2 把 GUI 截图转化为结构化元素（让任何 LLM 都能做 GUI Agent），Kimi-VL 内置 GUI 理解能力。从此，多模态模型的输出不再是文字描述——而是操作动作。

Kimi-VL（月之暗面, 2025.04） 是一个代表新路线的案例：MoE 视觉语言模型，仅激活 2.8B 参数即在多模态推理上达到强性能，自研 MoonViT 视觉编码器，内置 GUI 理解和 UI 操作功能。它代表了一种"轻量激活、高效推理"的设计路线。

格局的奠定：Qwen3-VL 在 MMMU 上达到 86.0%，超越所有闭源模型。InternVL 3.5（2025）通用感知能力继续提升，Mono-InternVL（CVPR 2025）向单体架构推进。Gemini 3.1 Pro 在 MMMU-Pro 82% 保持科学推理领先。

3.5 三条技术线演进的总结

回顾从 2021 到 2026 的进程，多模态大模型的演进可以归结为三条并行线：

线一——视觉编码：CLIP (ViT) → SigLIP → InternViT-6B → 动态分辨率。从固定尺寸到动态分辨率，从通用编码器到多模态专用编码器。

线二——模态连接：Flamingo 交叉注意力 → Q-Former → MLP（LLaVA）→ 视觉专家模块（CogVLM）。从复杂到极简再回归合理复杂度。

线三——训练范式：冻结编码器+冻结 LLM → 只训练连接器 → 全参数微调 → 渐进式训练。从"怕动参数"到"大胆全量调"。

2026 年的状态："后期拼接"和"原生多模态"两条路线依然并存，但差距在缩小。Qwen3-VL（后期拼接）在 MMMU 上超越 GPT-5.1（原生多模态）——说明"架构选择不再决定性能上限，规模和数据才是。"

4.3 三家闭源旗舰的深度分析

OpenAI GPT 系列：全模态通用选手

GPT-4o（2024.05）是第一个全模态统一的商用模型。它的核心差异化优势不是单项能力最强，而是所有模态都能做到 80 分以上——文本、图像、语音、代码，没有明显短板。

GPT-5.1（2025）在 MMMU 上达到 85.4%——推理能力的大幅提升让它在处理复杂的多模态推理任务时更有优势。API 生态依然是行业最成熟的，工具链（Function Calling、Assistants API、Structured Outputs）与多模态能力的整合最顺畅。

短板：长上下文不如 Gemini（标准 128K vs Gemini 的 2M），多模态输出能力有限（生成图像需要依赖 DALL-E 而非原生）。在特定垂直场景（如科学推理、精细 OCR）不是最强。

Google Gemini 系列：长上下文+科学推理之王

Gemini 1.5 Pro（2024.02）在长上下文多模态上建立了持续领先——1M token 的上下文不是噱头，而是能真正处理长视频、长篇文档、多图序列的实际能力。

Gemini 3.1 Pro（2026.02）在科学推理维度（Science Horizon multi-modal 88.85 分）全面领先。MMMU-Pro 82% 是所有模型中最高的——MMMU-Pro 是 MMMU 的强化版，需要更深层的推理能力。

Gemini 的真正优势在于 Google 的基础设施整合：从 YouTube 视频直接输入、与 Google Drive 文档的深度集成、2M token 上下文窗口。在"多模态 + 长上下文"的交叉场景（分析一个 1 小时的讲座视频并回答关于其中某个细节的问题），Gemini 没有对手。

短板：编程和写作质量在 Reddit 社区的反馈中略逊于 Claude。开发者生态不如 OpenAI 成熟。

Anthropic Claude 系列：精确度至上

Claude 系列在多模态上走了一条保守但有效的路线。Claude 3 Sonnet（2024.06）的 MMMU 68.3% 在发布时不如 GPT-4V，但它在图表理解、文档 OCR 上的精确度被广泛认可——模型不会"瞎编"图片中的细节，不会"过度解读"模糊内容。

Claude Opus 4.x（2025-2026）在 SWE-bench 上领先（82-87%），加上 Computer Use 能力（2024年底推出），使它在"屏幕理解 + 代码编写"的 Agent 场景上占据了独特定位。如果你是做一个需要"看屏幕→写代码→操作界面"的 Agent，Claude 是最优选择。

社区口碑（Reddit、X/Twitter）上，Claude 的标签是"最聪明也最贵"。在需要精细视觉分析的场景（错误排查、UI 测试、文档审核），用户会优先选择 Claude。

短板：MMMU 基准落后于 OpenAI 和 Google。上下文窗口（200K）不如 Gemini 的大。价格是三者中最贵的。后期拼接架构在多模态深度融合的理论上限可能不如原生多模态。

4.4 开源领跑者：Qwen3-VL 和 InternVL

Qwen3-VL 在 2025-2026 年成为开源多模态的绝对标杆。MMMU 86.0% 超越所有闭源模型，且价格极低（72B 约 $0.20/M input，7B 模型可本地部署）。Qwen3.5-Plus 在科学地平线多模态评分 84.56，仅次于 Gemini 3.1 系列。

Qwen 的成功在于兼顾了性能、价格和生态：动态分辨率 ViT 保证了文档/图表的处理能力，三层训练管线保证了模型质量，Apache 2.0 协议和积极的社区运营保证了开源生态。Hugging Face 上下载量最高，社区贡献最活跃。

InternVL2.5-78B 的视觉编码器最强（InternViT-6B），在 MMBench 等视觉细分任务上有时能超过 Qwen。但整体生态（推理框架支持、社区工具）不如 Qwen 成熟，部署成本也更高（6B + 78B）。

开源 vs 闭源的格局变化：2024 年还是"开源追赶闭源"，2025-2026 年已经是"开源在某些任务上超越闭源"。这不是因为开源突然有了更好的算法，而是因为开源模型的规模追上来了——Qwen3-VL-72B 的参数量级和训练数据规模已经和闭源旗舰处于同一水平。

关键判断修正：2024 年底的判断是"原生 vs 拼接差距已被规模稀释"。但 2025 年 Apple 的 Scaling Law 和 Llama 4 的路线投票让这个判断需要修正。

Apple 的核心发现是：早融合 + MoE 的 scaling law 斜率比后融合更陡——也就是说，随着模型规模扩大，"后期拼接"的天花板会更早出现。这个发现还没有被大规模实验完全验证，但它来自 Apple 的基础研究团队，不应被忽视。

商汤 NEO 的完整抛弃视觉编码器方案、Tuna-2 的"像素嵌入替代视觉编码器"——这些都在追问同一个问题："CLIP 视觉编码器是不是一个必要的恶？" 如果是，那它的最佳替代是什么？2026 年还没有确定答案，但这个问题本身已经开始定义下一个阶段的技术竞争。

五、横纵交汇洞察

5.1 CLIP 的遗产：对比学习的范式

纵向历史的起点（CLIP, 2021）锚定了一个范式：对比学习 + 弱监督数据 = 可迁移的视觉表征。这个范式直到 2026 年仍然是多模态模型的基石。

CLIP 的对比学习目标（把匹配的图文对拉近，不匹配的推远）被 SigLIP（sigmoid loss 替代 softmax）改进后，仍然是视觉编码器训练的核心方式。即便是 Qwen3-VL 和 GPT-5 的内部视觉编码器，也建立在对比学习的思想之上。

这个范式的持久性告诉我们一个行业规律："好的基础问题定义"比"花哨的架构创新"更有生命力。对比学习定义了正确的问题（"图文是否匹配"），而后续所有工作都是这个答案的不同实现。

5.2 Q-Former vs MLP：效率与性能的 trade-off 最终被规模抹平

纵向历史中的一个有趣节点是 2023 年的"连接器之争"：

  • Q-Former：188M 参数，用 query 从视觉特征中"提取"信息，压缩成少量 token
  • MLP（LLaVA）：~2M 参数，直接把视觉 token 映射过去

Q-Former 效率更高（更少 token → LLM 计算量更小），MLP 保留信息更多（更多 token → 细节保留更好）。

到了 2024-2025 年，当 LLM 的上下文窗口从 4K 扩展到 128K-1M，"token 多了怎么办"这个问题不再成立——LLM 已经能处理大量视觉 token。MLP 路线因此胜出。Q-Former 的"效率优势"被 LLM 上下文窗口的扩展直接抹平。

这个案例揭示了一个深层 trade-off：中间件的价值取决于主件的瓶颈。当 LLM 的上下文很贵时，Q-Former 有价值；当上下文变便宜时，Q-Former 的价值就稀释了。

5.3 开源追赶闭源的"时间窗口"越来越短

2023 年 9 月的 GPT-4V 到 2024 年 4 月的 InternVL 1.5——开源花了 7 个月追到接近水平。

2024 年 5 月的 GPT-4o 到 2024 年 9 月的 Qwen2-VL——开源花了 4 个月追到同等水平。

2025-2026 年：Qwen3-VL 在 MMMU 上超越闭源——开源在基准测试上正式领跑。

这个"追赶窗口"从 7 个月缩短到几乎同步。原因不是开源突然开窍了，而是多模态的架构已经收敛到"视觉编码器 + 连接器 + LLM"的标准范式——在这个范式下，计算的规模效应超过架构的设计差异。谁有更多的算力和数据，谁就能训练出更好的模型。而阿里（Qwen）和上海 AI Lab（InternVL）的算力投入已经不比闭源厂商少多少。

5.4 多模态的趋势：从"看懂"到"全模态"——以及原生多模态的回归

2021-2023：模型努力学会"看懂"一张图片。

2024：GPT-4o 统一文本+图像+音频，Gemini 统一文本+图像+视频。

2025-2026：多模态进入"全模态"阶段。模型不再区分"你是文本任务"还是"你是图像任务"——所有输入都被当作同一个模态空间中的序列来处理。

这个趋势的最终形态是：未来的大模型将没有"单模态版本"。文本-only 的模型将会像"不支持表格输入的数据库"一样古怪。

但 2025 年还撕开了一个更深的裂缝：原生多模态的回归。 这个回归的逻辑链是这样的——

CLIP（2021）建立了一个范式：视觉编码器独立训练（对比学习），然后通过连接器挂接到 LLM。这个范式在过去五年被证明极其高效——它复用了已有的 LLM，降低了训练门槛，催生了整个开源多模态生态。但 Apple 的 Scaling Law 研究（2025.04）给出了一个令人不安的答案：这个范式的性能天花板可能比人们想象的低。 早融合（从像素直接出发）在大规模训练下的性能斜率更陡，意味着当模型规模超过某个临界点时，后期拼接路线的边际收益会递减。

这是否意味着我们正在见证又一个"中间件被主件吞没"的案例——就像 Q-Former 被 LLM 上下文扩展抹平一样，视觉编码器本身会不会被"更深的训练"抹平？

2026 年还没有答案。Qwen3-VL（后期拼接）仍然是 MMMU 冠军，Tuna-2 和 NEO 的原生方案还在追赶中。但这股回归浪潮已经改变了行业对这个问题的判断方式——"CLIP+连接器"不再被视为不言自明的标准答案，而是一个需要被重新论证的选择。

5.5 三个未来剧本

剧本一：多模态成为基础能力，竞争转向 Agent（概率 70%）

多模态视觉能力不再是差异化因素——所有主流模型都能理解图片、处理文档。竞争焦点转移到"多模态 Agent"：模型能不能看屏幕、操作界面、执行代码、做端到端任务。Claude 的 Computer Use、GPT-4o 的实时语音对话、Gemini 的 Agent 框架将成为下一代竞争的核心。

剧本二：视频理解取得突破（概率 20%）

长视频理解（>30 分钟）在当前是几乎所有模型的短板——帧采样会丢失关键信息，全帧处理算力不够。如果 Ring Attention + 高效的时空编码取得突破，视频理解将成为下一个"上下文竞赛"级别的战场。这将兑现原生多模态路线的长期赌注。

剧本三：多模态幻觉问题引发信任危机（概率 10%）

如果多模态幻觉（模型"看到"不存在的物体、读错关键数字）不能得到有效控制，在高风险场景（医疗影像、法律文档）的落地将受阻。对比解码等技术正在缓解但未根治。信任问题可能成为多模态普及的隐形天花板。

5.6 一条暗线：训练数据的诅咒

多模态大模型的训练数据高度依赖互联网图文对。但互联网图文对的质量并不均匀——电商页面、社交媒体帖子、新闻配图占了大多数，而高质量的专业领域图文数据（医学影像报告、工程图纸、科学图表）稀缺且分散。

这导致当前的模型在通用场景上好用，但在专业垂直场景上可能"不够好"。2025-2026 年出现的一个趋势是垂直多模态模型——在医疗、法律、金融等领域的专业图文数据上做继续训练。这可能成为多模态领域的下一个增长点。
