大模型 AI 芯片格局前置知识

写给完全零基础的读者——读完这篇，你就能看懂横纵分析报告的主体内容。

1. GPU 和 CPU 有什么区别？

CPU（中央处理器） 就像一个大厨——他什么菜都会做，但一次只能做几道菜，做的过程很精细。

GPU（图形处理器） 就像一千个帮厨——每个人只会做切菜这一件事，但一千个人同时切，速度比大厨一个人快得多。

AI 训练的核心是"做几万次简单的数学运算（矩阵乘法）"，所以 GPU 比 CPU 快 100-1000 倍。

2. TPU 又是什么？和 GPU 有什么区别？

TPU（Tensor Processing Unit，张量处理单元）是 Google 专门为 AI 计算设计的芯片。

如果 GPU 是一千个帮厨（能做各种简单任务），TPU 就是一千个只切土豆丝的帮厨——做的事情更少，但切得更快、功耗更低。

所以 TPU 在 AI 推理场景下比 GPU 更省电、更便宜。但它的缺点是"只擅长 AI 计算"——干不了渲染、科学计算等其他活。

3. TFLOPS 是什么？

TFLOPS（Tera FLOPs，万亿次浮点运算/秒）是衡量芯片算力的单位。1 TFLOPS 就是一秒钟运算一万亿次。

但注意：数字大不一定代表更好。 因为不同精度下测出来的 TFLOPS 完全不同：

  • FP32（32位浮点）：精度最高，但算力最低
  • FP16/BF16（16位）：精度适中，算力提高约 2 倍
  • FP8（8位）：精度较低，算力再高约 2 倍
  • FP4（4位）：精度最低，算力再高约 2 倍

所以一个标称 1000 TFLOPS (FP16) 的芯片，在 FP32 精度下可能只有 125 TFLOPS。厂商通常报自己最强的那个数字。

4. Tensor Core 是用来做什么的？

Tensor Core 是 NVIDIA 2017 年在 GPU 里加入的专门硬件单元，用来加速深度学习中最频繁的计算——矩阵乘法。

普通 CUDA Core 一次算 1 个乘加运算，Tensor Core 一次算 16 个（4x4 矩阵乘法）。效率差了 16 倍。

这个区别可以用一个比喻理解：

  • CUDA Core = 一个人用手算加减乘除
  • Tensor Core = 一台小计算器，按一下就算出一整张表

5. HBM 是什么？

HBM（High Bandwidth Memory，高带宽内存）是 AI 芯片使用的一种特殊内存，堆叠在芯片旁边，通过微小通道高速传输数据。

想象一下：你有一个超快的计算器（GPU），但它要从一个超慢的文件柜（普通内存）里找数据——算得再快也没用，因为大部分时间在等数据送来。

HBM 就是把文件柜搬到计算器旁边，用几十条高速公路连起来。数据送达速度从"骑自行车"变成了"坐高铁"。

HBM 的规格（HBM2、HBM3、HBM3e）也决定了带宽——HBM3e 比 HBM3 快约 40%。

6. NVLink 和互联技术

当你用几千块 GPU 一起训练一个模型时，它们需要频繁地相互通信。"每块 GPU 的计算速度"不是瓶颈，"GPU 之间的通信速度"才是。

NVLink 是 NVIDIA 的私有互联技术，专门解决这个问题：

  • H100 的 NVLink 4：900 GB/s
  • B200 的 NVLink 5：1.8 TB/s

这个速度的意义：PCIe 4.0 x16 的带宽只有 32 GB/s。NVLink 比它快 30-60 倍。

7. 什么是 CUDA？为什么它这么重要？

CUDA 是 NVIDIA 在 2007 年推出的 GPU 编程平台，让开发者可以用代码控制 GPU 做通用计算（不只是渲染画面）。

CUDA 的重要性不在于技术，而在于生态。经过近 20 年积累：

  • 400 万 + 开发者用 CUDA 写过代码
  • 所有 AI 框架（PyTorch、TensorFlow、JAX）底层都调 CUDA
  • 大量优化库（cuDNN、cuBLAS、TensorRT）只有 CUDA 版本

cuDNN 是其中最关键的库——专为深度学习优化的 GPU 加速库。所有主流框架的 GPU 加速层都基于 cuDNN 构建。它的战略意义在于：开发者写的 PyTorch 代码底层全是 cuDNN 调用，换成 AMD 的 ROCm 就得重新适配。

这就导致了"CUDA lock-in"——你想换 AMD 的 ROCm，但你的代码库里全是 CUDA 调用，迁移成本极高。行业里有一句话："CUDA is not just a moat, it's an ocean."

7.5 MIG（多实例 GPU）

MIG（Multi-Instance GPU）是 NVIDIA 从 A100 开始引入的技术，把一块物理 GPU 切分成最多 7 个独立实例。每个实例拥有独立的内存和计算资源，互不干扰。

意义：云厂商可以像卖虚拟机一样卖 GPU 切片。你不需要租一整块 H100（每月数千美元），可以只租 1/7 块做小模型推理。这大大降低了 GPU 云服务的使用门槛。

8. 什么是 Chiplet？

传统芯片是把所有功能做在一大块硅片上（monolithic die）。Chiplet 是把芯片拆成几个小块，用高速互联组装起来。

好处：

  • 每小块良品率更高（大芯片面积越大、良品率越低）
  • 不同小块可以用不同制程（计算部分用 3nm，I/O 部分用 7nm 更省钱）
  • 设计复用（可以换不同小块来组合出新芯片）

B200 就是双 chiplet 设计（两个计算 die 通过 NV-HBI 互联，带宽 10 TB/s）。

9. ASIC 是什么？

ASIC（Application-Specific Integrated Circuit，专用集成电路）是"为特定任务设计的芯片"。

把市面上所有芯片放一个谱系上：

  • CPU（最通用，什么都能做，但效率最低）
  • GPU（通用性中，图形+计算两用，效率中等）
  • ASIC（最专用，只能做一件事，但效率最高）

TPU 就是 ASIC。它的优势是推理时效率极高（Midjourney 切到 TPU 后成本降低了 65%），劣势是"不能做别的"。

10. 出口管制如何影响 AI 芯片？

2022 年 10 月起，美国禁止向中国出口高端 AI 芯片（A100/H100）。这带来了几个连锁反应：

  • NVIDIA 推出降级版：H800（降互联带宽）→ H20（大幅降级）→ 最终也被禁
  • 中国被迫自己造：华为昇腾 910B 成为替代方案，但性能差距明显
  • "两个生态"正在形成：西方用 NVIDIA/AMD/TPU，中国用昇腾等国产芯片
