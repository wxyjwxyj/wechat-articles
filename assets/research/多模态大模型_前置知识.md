# 多模态大模型 前置知识

> 这份文档为零基础读者准备。如果你已经熟悉 CLIP、Q-Former、视觉编码器等概念，可以直接翻阅主报告。

---

## 一、什么是多模态大模型？

多模态大模型（Multi-modal Large Language Model, MLLM）是**能同时理解和处理多种信息类型（文本、图像、视频、音频）的大模型**。

普通 LLM 只能读写文字。你把一张照片给它，它"看不见"。多模态大模型不一样——它能"看"图片、"听"音频、"读"图表，并且结合文字理解来做推理。

### 举个具体的例子

```
你给模型一张照片：一只猫坐在笔记本电脑旁边
你问："这只猫在干什么？"
普通 LLM：看不懂图片，只能回答"我没有办法看到图片"
多模态 LLM："这只猫坐在一台打开的笔记本电脑旁边，看起来像是在'用电脑'"
```

### 关键能力

- **图片问答**：分析照片、图表、截图、漫画
- **OCR / 文档理解**：读出图片中的文字、理解表格和合同
- **视频理解**：看懂一段视频的内容
- **多图推理**：比较两张图的异同

## 二、多模态模型是怎么"看"的？

一个直觉模型：多模态大模型 = 眼睛（视觉编码器）+ 翻译器（连接器）+ 大脑（LLM）。

**视觉编码器（眼睛）**：把图片拆成小片（patch），每个小片转换成一个向量（token）。一张 448x448 的图片被拆成 1024 个 patch，每个 patch 变成一个 token。最常用的编码器是 ViT（Vision Transformer）。

**连接器（翻译器）**：把视觉 token 翻译成 LLM 能理解的"语言"。最简单的是 MLP（几层神经网络），复杂一点的有 Q-Former（一个专门的 Transformer）。

**LLM（大脑）**：把文本 token 和视觉 token 放在一起处理，进行推理和回答。

```
图片 → [视觉编码器] → 视觉 token → [连接器] → LLM 能理解的 token → [LLM] → 回答
"这张图里有什么？" → [分词] → 文本 token → ------------------------↑
```

## 三、CLIP——多模态的"基础语言"

CLIP（OpenAI, 2021）是多模态领域的奠基之作。它的核心理念非常简单：

**用 4 亿张图文对照的图片来训练一个模型，让它学会"一句话和一张图是不是在说同一件事"**。

训练数据格式：
```
[图片: 一只金毛在草地上奔跑] ← 匹配 → "一只金毛在草地上奔跑"
[图片: 一只金毛在草地上奔跑] ← 不匹配 → "一辆红色汽车停在路边"
```

训练完成后，CLIP 学会了把"语义相近"的图片和文字在数学空间中放在很近的位置。这就是 Contrastive Learning（对比学习）——核心直觉：配对的正样本拉近，不配对的负样本推远。

CLIP 的贡献在于：它第一次证明了**不需要人工标注，直接用互联网上的图文对就能训练出强大的视觉理解能力**（这被称为"弱监督"——监督信号比人工标注弱，但数据量大得多）。后来的多模态模型几乎都建立在 CLIP 或它的改进版本之上。

### SigLIP——CLIP 的改进版

Google 在 2023 年提出的 SigLIP 改进了 CLIP 的训练方式。CLIP 需要在一个 batch 内做全局归一化（对比所有图文对的相似度），batch 小了效果就差。SigLIP 换了一种损失函数（sigmoid loss），逐对独立计算，小 batch 训练同样有效。到 2026 年，SigLIP 已成为新版多模态模型的首选视觉编码器。

## 四、Q-Former——高效"翻译官"

Q-Former（Salesforce, 2023）是 BLIP-2 的核心创新，解决了一个关键问题：怎么把大量视觉信息"压缩"成 LLM 能高效处理的少量 token。

### 问题

一张图片经过 ViT 编码后会产生几百到几千个视觉 token。如果全部送进 LLM，序列太长、计算量太大。

### Q-Former 的解决方案

引入一组"可学习的查询向量"（learnable queries），比如 32 个。这些 query 通过交叉注意力机制从视觉编码器的输出中"提取"关键信息。

**直觉**：Q-Former 像是一个助手，它拿着一张"问题清单"（32 个查询），仔细扫描视觉编码器的输出，把最相关的信息摘录下来，然后浓缩成简短的摘要递给 LLM。

### 为什么这重要？

因为 LLM 的上下文窗口有限，计算量随序列长度平方增长。Q-Former 把几百个视觉 token 压缩成 32 个，大幅降低了计算成本。

## 五、原生多模态 vs 后期拼接

这是多模态领域最重要的技术路线分歧。

### 后期拼接（主流方案）

先把 LLM 在纯文本上训练好，再通过连接器把视觉能力"接"上去。

**代表**：LLaVA、Qwen-VL、InternVL、Llama 3.2 Vision

**优点**：实现简单、可以利用已有的文本 LLM、训练成本低
**缺点**：视觉和语言的对齐不够深入

### 原生多模态

从模型设计之初就把所有模态放在一起训练。

**代表**：Gemini、GPT-4o

**优点**：模态间交互更深、理论性能上限更高
**缺点**：训练成本极高、技术门槛高

打个比方：后期拼接是给一个已经毕业的法律专家配一副眼镜让他能看证据材料；原生多模态是培养一个从入学就同时学法律和取证分析的学生。

## 六、动态分辨率——看清细节的关键

早期多模态模型把图片全都缩放到固定大小（如 336x336）再处理。但这样会丢失细节——合同上的小字、发票上的数字都看不清楚。

**动态分辨率**（Dynamic Resolution）的解决方案：不缩放图片，而是根据图片的实际尺寸动态决定处理方式。对于一张高分辨率文档图片，把它切成多个子图块（tiles），每个子图单独编码再拼起来。

Qwen2-VL 的做法是"保留原始宽高比，按原生分辨率处理"。一张 2048x2048 的图片会被切成多个 448x448 的 tile，每个 tile 独立编码。这大幅提升了 OCR 和文档理解能力。

## 七、多模态幻觉——模型在"瞎编"

跟普通 LLM 的幻觉问题类似，多模态模型也会"看到不存在的东西"。

最常见的类型：
- **对象幻觉**：说图片里有不存在的东西（"这张图里有一辆车"——其实没有）
- **属性幻觉**：物体的颜色、大小、位置说错（"猫是白色的"——其实是橘色的）
- **关系幻觉**：物体之间的关系说错（"狗在猫的左边"——实际在右边）

根源往往是 LLM 的"语言先验"太强——模型"猜"出常见的搭配（"草地上通常有狗"），而不是真的从图片中看到了狗。

## 八、多模态评测——怎么衡量好不好

| 评测 | 测试什么 | 简单理解 |
|------|---------|---------|
| **MMMU** | 多学科理解 | 给模型看含图的高考/大学题目，看能不能答对 |
| **MMBench** | 通用多模态能力 | 涵盖 20 种视觉能力的综合测试 |
| **MathVista** | 数学推理 | 带图的数学题 |
| **ChartQA** | 图表理解 | 看柱状图/折线图回答问题 |
| **OCRBench** | 文字识别 | 识别图片中的文字 |
| **Video-MME** | 视频理解 | 看懂视频内容 |

## 九、一张图总结

```
多模态大模型 = 眼睛 + 翻译器 + 大脑

          ┌─────────────────────────────────────┐
          │             多模态大模型              │
          │                                     │
 图片 →   │  视觉编码器 (ViT/SigLIP)             │
          │      ↓                               │
          │  连接器 (MLP/Q-Former)               │
          │      ↓                               │
 文字 →   │  LLM (理解 + 推理 + 回答)            │  → 回答
          │                                     │
          └─────────────────────────────────────┘

 技术路线分歧：
   后期拼接 (LLaVA)  vs  原生多模态 (Gemini)
     简单便宜                 性能上限高
     可复用已有 LLM           训练成本极高

 关键挑战：
   • 分辨率 vs 成本（越清晰 token 越多）
   • 多模态幻觉（瞎编图片里没有的东西）
   • 视频理解（帧太多，算不过来）
```
