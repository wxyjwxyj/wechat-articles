机器学习概览横纵分析报告

从"让机器从数据中学习"的基本思想出发——监督学习、无监督学习、强化学习三大范式，以及评估模型好坏的完整框架。

一、一句话定义

机器学习（Machine Learning）是让计算机从数据中自动学习规律和模式，而非通过显式编程来执行任务的方法论。它的核心范式转变是：不是告诉机器"如果看到 X 就做 Y"，而是给机器一堆（X, Y）的例子，让它自己找出"X → Y"的映射规律。

🎯 读完这篇你能：区分监督/无监督/强化学主要范式的适用场景，理解过拟合、正则化、交叉验证等核心概念，能判断"这个 ML 模型的问题出在哪里"。

二、三大学习范式

2.1 监督学习 —— 有标准答案的学习

定义：训练数据中每个输入 X 都有对应的正确答案 Y（标签）。

类比：老师给了很多例题和答案，学生学的是"这种题型→这种解题方法"的规律。

核心算法：

  • 线性回归：预测连续数值（房价预测：面积→价格）
  • 逻辑回归：二分类（垃圾邮件检测：邮件文本→是/否垃圾）
  • 决策树/随机森林：基于特征的规则判断（贷款审批：收入+信用分+年龄→批准/拒绝）
  • SVM（支持向量机）：找最优分类边界（图像分类、文本分类）
  • XGBoost/LightGBM：梯度提升树——Kaggle 竞赛和工业界表格数据的"瑞士军刀"

评估指标：

  • 分类：准确率、精确率、召回率、F1 分数、AUC-ROC
  • 回归：MAE（平均绝对误差）、MSE（均方误差）、R²

2.2 无监督学习 —— 没有标准答案，自己找结构

定义：训练数据没有标签，模型要自己发现数据中的隐藏结构。

类比：给一堆照片但不告诉你是谁，你自己按长相相似度分成几组——虽然不知道每组的"名字"，但你发现了数据的自然分组。

核心方法：

  • 聚类（Clustering）：K-Means、DBSCAN、层次聚类——把相似的数据点归为一组
  • 降维（Dimensionality Reduction）：PCA、t-SNE、UMAP——把高维数据压缩到 2-3 维做可视化
  • 异常检测（Anomaly Detection）：找出"不合群"的数据点（信用卡欺诈检测）
  • 关联规则（Association Rules）：Apriori——发现"买了 A 的人通常也买 B"的规律

2.3 强化学习 —— 在实践中学习

定义：Agent 在环境中尝试动作，根据奖励信号学习最优策略。没有"标准答案"，只有"奖励/惩罚"。

类比：训练狗——你不需要告诉它每个命令的具体执行步骤，只需要在它做对时给零食、做错时不给。

详见 大模型强化学习横纵分析报告。

核心概念：

  • Agent / Environment / Action / State / Reward / Policy
  • Exploration vs Exploitation：探索新策略 vs 利用已知好策略——强化学习的核心矛盾

三、机器学习的通用难题

3.1 过拟合（Overfitting）—— 背住了题，没学会解法

训练集上 99% 准确率，测试集上 70%——这是典型的过拟合。

本质：模型学会了训练数据中的噪声和特例，而非真正的底层规律。

解决方案：

  • 正则化（Regularization）：L1（产生稀疏参数）/ L2（限制参数大小）——给模型加"不要太复杂"的约束
  • Dropout：训练时随机丢弃部分神经元，强迫每个神经元都学会有用特征
  • 早停（Early Stopping）：验证集误差开始上升时立刻停止训练
  • 更多数据：最根本的解决办法（但在 LLM 时代，"更多数据"本身就有取舍）

3.2 偏差-方差权衡（Bias-Variance Tradeoff）

  • 高偏差（欠拟合）：模型太简单，连训练集都拟合不好。解决方案：加复杂度
  • 高方差（过拟合）：模型太复杂，训练集效果极好但泛化差。解决方案：加约束/加数据

3.3 特征工程 vs 表示学习

  • 传统 ML：特征工程是最重要的工作——人类专家花 80% 时间设计和选择特征
  • 深度学习：网络自动从数据中学习特征表示——"让数据自己说话"

这个转变是深度学习革命的核心——不是算法变得更聪明了，而是特征学习的负担从人转移到了机器。

四、评估体系：怎么知道模型好不好？

方法怎么做目的
训练/验证/测试集划分训练集练模型，验证集调超参，测试集最终考核防止"考试作弊"（用测试集调参）
交叉验证把数据分成 K 份，轮流留一份做测试评估稳定性
混淆矩阵统计 TP/FP/FN/TN看模型在哪类错误上最多
ROC/AUC画不同阈值下的 TPR vs FPR评估排序能力

五、从经典 ML 到大模型的范式转变

维度经典 MLLLM 时代
特征工程人工设计（80% 工作量）模型自动学习
训练方式每个任务训练一个模型一个预训练模型适配所有任务
数据需求几百到几万标注样本万亿 token 预训练 + 少量标注
泛化方式同分布假设Few-shot / Zero-shot 跨任务泛化
可解释性较高（特征权重可见）较低（黑盒）
部署小模型，CPU 可跑大模型，需要 GPU/NPU

经典 ML 并没有被淘汰——XGBoost 在表格数据上仍然比 LLM 更高效；K-Means 在聚类上仍然比 LLM 快千百倍。经典 ML 和 LLM 是互补关系，不是替代关系。
