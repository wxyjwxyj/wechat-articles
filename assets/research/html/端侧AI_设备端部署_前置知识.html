端侧 AI / 设备端部署前置知识

把 AI 塞进手机和笔记本——为什么、怎么做到的、有什么取舍。

一、什么是端侧 AI？

端侧 AI（On-Device AI）就是把 AI 模型直接运行在用户设备上（手机、PC、平板、IoT 设备），而不是通过网络调用云端服务器。

用快递来比喻：

  • 云端 AI = 你打电话给中央厨房，厨房做好菜派快递送过来。好吃，但要等，还要付配送费。
  • 端侧 AI = 你自己在家做菜。食材有限（小模型），但随做随吃，不用等，不用配送费，而且没人知道你今天吃了什么（隐私）。

二、为什么要搞端侧 AI？三大驱动力

1. 隐私：你的聊天记录、照片、文档不需要离开设备就能被 AI 处理。Apple 把"隐私"作为端侧 AI 的核心卖点。

2. 延迟：云端 API 的网络往返通常 200-1000ms，而端侧推理可以做到 10-50ms。语音助手说"打开相机"——你等不了 500ms，你需要立刻响应。

3. 成本：每次云端 API 调用都要付费。端侧推理的边际成本为零（电费忽略不计）。如果你每天用 AI 100 次，端侧一年省下的 API 费用可能够买一台新手机。

4. 离线：飞机上、地铁里、地下室——端侧 AI 不需要网络连接。

三、端侧 AI 怎么做到的？四个关键技术

3.1 专用 AI 芯片（NPU）

NPU = Neural Processing Unit，专门为神经网络推理设计的芯片。

CPU（中央处理器）是"全能选手"——什么都能做，但 AI 推理不是最优。GPU（图形处理器）擅长并行计算，做 AI 推理比 CPU 快很多，但功耗大。NPU 是"AI 专家"——专门为矩阵乘法（神经网络的核心运算）优化，同等算力下功耗只有 GPU 的 1/3 到 1/5。

2026 年旗舰设备的 NPU 算力：

  • Snapdragon X2 Elite：80 TOPS（每秒 80 万亿次运算）
  • Apple M4 Neural Engine：38 TOPS
  • Intel Panther Lake：100+ TOPS

3.2 模型压缩（量化）

量化 = 把模型参数从高精度（FP16，每个参数 16 bit）压缩到低精度（INT4，每个参数 4 bit）。

直觉：参数 16.374 和 16.3 在大多数情况下对结果的影响可以忽略。量化就是利用这种"精度冗余"——一个 7B 的 FP16 模型需要 14GB 显存，量化到 4-bit 后只需要约 4GB。

3.3 小模型（SLM）

不是把大模型硬塞进手机，而是从头设计能在手机上跑的小模型。详见 小语言模型前置知识。

3.4 推理引擎优化

llama.cpp、ONNX Runtime、Apple Core ML——这些运行时专门为端侧硬件做了优化，包括内存管理、算子融合、KV Cache 压缩等。

四、关键概念

概念一句话
TOPSTera Operations Per Second——衡量 AI 芯片算力的单位
NPU专门为神经网络设计的处理单元，比 GPU 更省电
端侧推理在用户设备上运行模型（不需要网络）
混合 AI简单任务端侧做、复杂任务云端做，用户无感知切换
量化把模型参数从 FP16 压到 INT4，内存需求减半再减半
内存带宽端侧 AI 的真正瓶颈——不是算力不够，是数据不够快
