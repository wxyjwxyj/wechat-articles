RoPE 旋转位置编码 前置知识

面向了解 Transformer 基本结构（知道 Q、K、V、Self-Attention）的读者。
🎯 读完这篇你能：理解为什么需要位置编码、RoPE 为什么比之前的方法好、能说出 RoPE 变体（PI/NTK/YaRN）各自做了什么。

一、为什么需要位置编码？

Transformer 的 Self-Attention 有一个天生缺陷：它不知道 token 的顺序。

"我打你"和"你打我"，每个 token 两两之间的注意力分数是完全一样的——Attention 只看内容，不看位置。但顺序显然很重要。

位置编码就是给每个 token 打上"我在第几个位置"的标签。

二、RoPE 之前：三种老方案

1. Sinusoidal（Transformer 2017）：用正弦/余弦函数生成固定模式的位置向量，直接加到 token embedding 上。不占参数但不能学习。

2. Learned（BERT/GPT 2018）：给每个位置一个可学习的向量，训练时自动调。问题是长度固定——GPT-2 训了 1024 个位置，给第 1025 个 token 就报错。

3. ALiBi（2021）：在 attention score 上加一个简单的线性惩罚——两个 token 距离越远扣分越多。极简，但表达能力有限。

三、RoPE 的核心想法：用旋转代替位移

苏剑林的洞察来自一个数学观察：把两个向量分别旋转后做内积，结果只依赖于它们旋转角的差，而不是各自的绝对角度。

所以他把 Q 和 K 的每两个维度看成一个"2D 向量对"，按位置给它旋转一个角度：

  • 位置 3 的 token，Q 旋转 3θ
  • 位置 7 的 token，K 旋转 7θ
  • Q 和 K 做内积时，结果只依赖于 (7-3)θ = 相对距离 4

这样既保持了绝对位置编码的便利性（直接作用在向量上），又获得了相对位置编码的自然外推能力。

直觉类比：你站在转盘上，你和你朋友分别转不同的角度——你们俩说话（做内积），内容只取决于你们之间的"相对角度差"，而不是各自面向的具体方向。

四、RoPE 为什么赢了？

2023 年 2 月，Meta 在设计 LLaMA 时对比方案后选了 RoPE。LLaMA 一成功，所有基于 LLaMA 微调和衍生的模型自动继承 RoPE。网络效应一旦形成就不可逆——FlashAttention、vLLM、llama.cpp 都对 RoPE 做了极致优化。

到 2026 年，RoPE 是 LLM 位置编码的事实标准。不用 RoPE 才是例外。

五、RoPE 的扩展：如何突破训练长度？

原版 RoPE 在训练长度（如 2048）外推理时效果急剧下降。几个关键扩展方案：

方案提出思路一句话
PIMeta 2023.06把所有旋转角等比例缩小"把位置也压缩进去"
NTK-aware社区 2023.07高频少缩、低频多缩"近处别动、远处多看"
Dynamic NTK社区 2023.07根据实际长度动态调整"多长就按多长的缩"
YaRN2023.09PI + NTK + 温度调优"三合一，当前最强"
LongRoPE2微软 2025进化搜索最优频率"近乎无损到 128K+"

六、关键人物

苏剑林（Jianlin Su），独立研究者，个人博客"科学空间"（kexue.fm）。2021 年在博客上发表了 RoPE 的推导思路，同时上传 arxiv。这是他一个业余项目的成果——没有大厂资源、没有 GPU 集群。但他的数学直觉打动了 Meta 的 LLaMA 团队。此后所有主流 LLM 都用 RoPE。

这是"一个人的一篇论文，穿透了整个行业"的故事。
