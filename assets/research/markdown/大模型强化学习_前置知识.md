# 大模型强化学习（RL）前置知识

> 面向零基础读者，用比喻和例子讲清楚 RL 在 LLM 中的核心概念。

---

## 一、什么是强化学习？

强化学习（Reinforcement Learning, RL）是机器学习的一种范式：**一个 Agent 在环境中尝试动作，根据得到的奖励来学习"什么动作在什么情况下最好"。**

用训狗来比喻：你教狗坐下——它乱跑（动作），你说"不对"（负奖励）；它坐下了（动作），你给零食（正奖励）。狗不需要你告诉它"怎么坐"，它只需要知道"坐下有零食"。RL 完全一样——你不需要告诉模型"怎么做"，只需要告诉它"做对了没有"。

这与监督学习（SFT）的根本区别：
- **SFT**：给模型看"标准答案"，让它模仿
- **RL**：让模型自己探索，根据奖励学习最优策略

## 二、RL 在 LLM 中的角色演变

### RLHF（2022）- 让模型"说人话"

RLHF = Reinforcement Learning from Human Feedback（基于人类反馈的强化学习）。

三阶段流程：
1. **SFT**：人类标注员写示范回答，模型学会"问答"格式
2. **RM 训练**：人类对不同回答排序，训练一个"奖励模型"来预测人类的偏好
3. **PPO 优化**：用奖励模型打分，通过 PPO 算法更新模型参数

关键是第 3 步要同时加载 4 个模型：策略模型（要训练的）、参考模型（防止跑偏）、价值模型（估计当前状态好坏）、奖励模型（打分）。工程代价巨大。

### DPO（2023）- 不需要奖励模型了

Direct Preference Optimization 的核心洞察：偏好数据中蕴含的对比信息可以直接优化策略，不需要先训练一个单独的奖励模型。

效果：训练中从 4 个模型降到 2 个模型，GPU 需求从 8 张降到 2 张。个人开发者也能做对齐了。

但 DPO 是**离线**的——只在已有偏好数据上优化，无法发现比数据中更好的回答。

### GRPO（2024）- 连价值模型也不要了

Group Relative Policy Optimization 做了最优雅的减法：去掉价值模型（critic）。

怎么做到的？对同一个 prompt 生成多个回答（比如 4 个），用这个 group 内的**相对分数**来替代绝对价值估计。"比其他答案更好"= 好，"比其他答案更差"= 差。不需要知道"4.3 分"这个绝对值。

DeepSeek-R1 的成功证明：去掉 critic、去掉 reward model，只用 GRPO + 可验证奖励（对/错），模型就能自己学会推理。

### RLVR（2025-2026）- 打破数据瓶颈

RLVR = RL with Verifiable Rewards（可验证奖励的强化学习）。

前面的方法都依赖人类来判断"好不好"——这是瓶颈，因为人工标注贵且慢。RLVR 的突破在于：**用可自动验证的规则替代人类判断。**

哪些是可以自动验证的？
- 数学题：答案对还是错 → 自动判断
- 代码：测试用例是否通过 → 自动判断
- Agent 任务：任务完成与否 → 自动判断

RLVR 让 RL 训练可以 24×7 无人值守、无限规模化。

## 三、关键概念速查

| 概念 | 一句话 | 重要性 |
|------|--------|:------:|
| Policy（策略） | 模型本身——给定输入，输出什么 | 核心 |
| Reward（奖励） | 对输出的评价信号——好还是坏 | 核心 |
| Rollout（展开） | 训练中让模型生成回答的过程 | 高 |
| Critic（价值网络） | 估计"当前状态有多好"的辅助模型 | 中（GRPO 去掉了） |
| Reward Model（奖励模型） | 预测人类偏好的辅助模型 | 中（DPO 去掉了） |
| On-Policy | 用模型自己生成的数据来训练 | 高 |
| Off-Policy | 用预先收集的数据来训练 | 高 |
| KL Penalty | 约束——防止模型"放飞自我"偏离太远 | 高 |
| Reward Hacking | 模型学会了"骗取高分"而非"真正做好" | 警惕 |
