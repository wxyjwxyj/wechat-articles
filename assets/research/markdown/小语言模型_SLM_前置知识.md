# 小语言模型（SLM）前置知识

> 为什么 14B 能打 70B，3B 能在手机上跑——小模型背后的核心逻辑。

---

## 一、什么是小语言模型？

SLM = Small Language Model，参数量通常在 100M-30B 之间的紧凑型语言模型。

对比：GPT-5.5 的参数量估计在万亿级别，而 Phi-4 是 14B，差了约 100 倍。但 Phi-4 在 MMLU 上达到了 84%，GPT-5.5 大约是 93%——100 倍的参数差距只带来了约 9 个百分点的提升。

这个事实是 SLM 存在的根本理由：**大部分场景不需要最强的模型，需要的是"够好且便宜"的模型。**

---

## 二、为什么小模型能逼近大模型？

### 2.1 参数冗余假说

大量研究指向同一个结论：大模型的绝大部分参数是冗余的。

证据链：
- **LoRA**：只训练 0.01% 的参数就能达到全量微调 95-99% 的效果
- **SparseGPT**：可以移除 50% 的参数而不显著影响质量
- **DARE**：随机丢弃 99% 的微调 delta，模型仍能恢复原能力

### 2.2 数据质量 > 参数数量

Microsoft Phi 系列的核心哲学：**用大模型生成高质量合成数据，用这些"教科书级"的数据训练小模型。**

直觉：10 本精心编写的教科书 > 1000 本粗制滥造的网文。Phi 团队用 GPT-4 生成"教科书质量"的数据，1.3B 的模型在编程任务上超越了 12B 模型。

### 2.3 推理能力可以蒸馏

DeepSeek R1-Distill 证明了：**大模型的推理能力可以通过数据传递。** 用 R1 671B 生成包含完整推理链的数据，训练一个 7B-32B 的小模型，小模型在数学推理上能达到大模型 80-95% 的水平。

### 2.4 MoE 让小模型"按需激活"

Mixture of Experts（混合专家）：模型有很多"专家模块"，但每次推理只激活少数几个。Qwen 3.6-35B-A3B 有 35B 总参数，但只激活 3B——在 3B 的推理成本下获得 ~30B 的性能。

---

## 三、关键概念

| 概念 | 一句话 |
|------|--------|
| 稠密模型（Dense） | 每次推理激活全部参数。简单但"贵" |
| MoE 模型 | 每次推理只激活部分专家。用"总参数量"换"激活参数量" |
| 合成数据 | 用大模型生成训练数据。SLM 训练的主要数据来源 |
| 推理链蒸馏 | 让 student 学习 teacher 的"思考过程"而不只是最终答案 |
| 量化 | 把 FP16 权重压缩到 4-bit。SLM 能在手机上跑的关键技术 |
| 推理成本 | 云端 API 按 token 收费；端侧推理成本为零 |

---

## 四、SLM 的局限性

- **通用知识广度**不如大模型——14B 不可能记住跟 1T 一样多的世界知识
- **复杂推理上限**不如大模型——AIME 级别的数学题还是需要大模型
- **长程 Agent 规划**不如大模型——需要强推理和错误恢复的多步任务
- **创意发散**不如大模型——小模型的输出更"规矩"但缺少惊喜
