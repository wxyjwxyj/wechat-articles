# 大模型蒸馏（Knowledge Distillation）前置知识

> 让大模型当老师教小模型——蒸馏为什么是"制造智能"而非"压缩模型"。

---

## 一、什么是知识蒸馏？

知识蒸馏（Knowledge Distillation）就是**让一个小"学生"模型学习一个大"老师"模型的输出，从而以更少的参数量获得接近老师的能力。**

类比人类教学：不是把百科全书塞给学生——而是让最好的老师（大模型）出题、解题、展示思考过程，学生跟着学。

---

## 二、蒸馏的四种方式

### 2.1 Logit 蒸馏（Hinton, 2015）

Logit 是模型输出层在 softmax 之前的原始数值。大模型输出 "猫: 8.2, 狗: 2.1, 兔子: 0.5"——不仅告诉你"这是猫"，还告诉你"狗比兔子更像猫"。

学生模型学习老师的这个"概率分布"而不只是"正确答案"。这比直接学硬标签效果好得多——因为分布里蕴含了老师从海量数据中学到的"相似性知识"。

- 优点：信息量大
- 局限：需要访问老师的内部 logits（白盒），闭源 API 做不到

### 2.2 数据蒸馏（2023-2025）

用老师模型生成训练数据，拿这些数据做 SFT 训练学生。

不需要访问 logits——只需要老师的 text output。闭源 API（GPT-4、Claude）也可以用。

这就是 Microsoft Phi 系列和 DeepSeek R1-Distill 使用的路线。关键不在"复制老师的答案"，而在于**让老师生成高质量的推理过程，学生学会的是"怎么想"而不是"答案是啥"。**

- 优点：黑盒可用，质量可筛选
- 局限：离线——学生不能主动问老师新问题

### 2.3 Online / On-Policy 蒸馏（2025-2026）

学生自己生成回答 → 送给老师打分 → 根据分数更新参数 → 再生成、再问老师。

与数据蒸馏的关键区别：**学生可以主动探索**——它能尝试老师训练数据中没出现过的新问题、新解法。这恢复了 RL 的探索能力，但比 RL 更稳定。

- 优点：在线探索，上限更高
- 局限：需要频繁调用 API（成本较高）

DeepSeek V4 的 OPD（On-Policy Distillation）就是这个路线的最新成果。

### 2.4 Self-Distillation（2026）

不需要外部老师。学生通过"练习→自我评价→改进"的循环，用自己的高质量版本来训练下一版。

- 优点：完全不需要老师 API，成本最低
- 局限：能力上限受限于自身

## 三、关键概念

| 概念 | 一句话 |
|------|--------|
| Teacher / Student | 老师模型（大）教学生模型（小） |
| Soft Target | 老师的概率分布（"80% 猫, 15% 狗, 5% 兔子"） |
| Hard Target | 标准标签（"猫"） |
| Dark Knowledge | Hinton 的词——soft target 中蕴含的"相似性信息" |
| Logits | softmax 之前模型输出的原始数值 |
| 数据蒸馏 | 用老师生成的 text 数据训练学生 |
| On-Policy | 学生用自己生成的数据去问老师，实时学习 |
| Off-Policy | 学生用老师预先生成好的固定数据学习 |
| OPD | On-Policy Distillation——DeepSeek V4 使用的多专家在线蒸馏 |
| 推理链蒸馏 | 蒸馏的不是"答案"而是"思考过程" |

---

## 四、蒸馏 vs 其他压缩方法

| 方法 | 压缩的是什么 | 需要老师？ | 压缩比 |
|------|------------|:--------:|:-----:|
| 量化 | 参数精度（FP16→INT4） | ❌ | 4x |
| 剪枝 | 参数数量（去掉冗余权重） | ❌ | 2x |
| 蒸馏 | 能力（知识、推理方式） | ✅ | 10-100x |

量化减少的是"每个参数占多少内存"，蒸馏减少的是"需要多少个参数"。蒸馏的压缩比远大于量化和剪枝。
