## 2. Claude Mythos 核心架构被 22 岁小伙逆推开源

**来源：** 新智元  
**关键词：** Anthropic、DeepSeek、开源、大模型架构

22 岁的 Kye Gomez（Swarms 创始人）用第一性原理，从公开论文推导出 Claude Mythos 的核心架构，并以 OpenMythos 全开源。

核心是「循环深度 Transformer」（RDT）：不堆参数堆循环，同一套权重最多跑 16 次，每次激活不同的 MoE 专家子集。推理全程在潜空间完成，不像 CoT 那样逐步输出中间 token。

三段式架构：Prelude（编码）→ Recurrent Block（循环核心，最多 16 轮）→ Coda（输出）。MoE 设计借鉴 DeepSeek-MoE，注意力机制用 DeepSeek-V2 的多潜变量注意力，KV 缓存压缩 10-20 倍。

实验数据：770M 参数的 RDT 追平 1.3B 标准 Transformer，参数效率翻倍。

**核心洞察：** Scaling 法则可能正在改写——未来最强的模型不是参数最多的，而是「想得最多次」的。Dario Amodei 预测中国 12 个月内将复刻 Mythos 级别能力。

## 3. 一个 CLAUDE.md 霸榜 GitHub 第一，蒸馏自 Karpathy

**来源：** 新智元  
**关键词：** Claude Code、AI 编程、开源

一个 Markdown 文件拿到 61.6k star，连续三天 GitHub Trending 第一。

华人开发者 Jiayuan Zhang 把 Karpathy 关于 AI 编程 Agent 的观察蒸馏成 4 条核心原则：

1. **编码前先思考** — 不确定时必须停下来问，不能猜
2. **简约至上** — 没被要求的功能不写，只用一次的代码不建抽象层
3. **精确编辑** — 只动被要求动的部分，匹配项目已有代码风格
4. **目标驱动** — 给验收标准而非具体步骤，让 AI 自己循环执行

Karpathy 本人的观察：编程工作流已从「80% 手写 + 20% AI」翻转为「80% Agent + 20% 修补」。他也警告了「Slopacolypse」——2026 年将涌出大量 AI 生成的低质量内容。

**值得关注：** 这标志着「agentic engineering」作为一个工程学科正在成形。核心不是模型能力，而是如何用结构化指令管住 AI 的行为边界。跟我们项目的 CLAUDE.md 实践完全一致。

## 4. Mythos 架构逆推开源（量子位视角）

**来源：** 量子位  
**关键词：** Anthropic、DeepSeek、循环 Transformer

与第 2 篇报道同一事件，但量子位补充了更多技术细节：

- 循环 Transformer 的两个关键实验（俄亥俄州立大学）：
  - **系统性泛化**：训练时没见过的知识组合，循环 Transformer 照样答对，标准 Transformer 失败

## 5. 苏度科技具身首秀：0 真机数据，98% 首次抓取成功率

**来源：** 量子位  
**关键词：** 具身智能、创业、Sim2Real

估值 20 亿美元的苏度科技发布首个模型 #Sudo R1，核心亮点：

- **纯仿真训练，零真机数据**，zero-shot 首次抓取成功率约 98%，两次尝试内接近 100%
- 60 分钟不间断抓取 200+ 次，覆盖透明、柔性、反光、不规则物体，不同光照和干扰条件
- 世界模型与强化学习一体化设计，打破了行业对 Sim2Real 路径的根本性质疑

技术路线与主流（PI、Generalist 的 few-shot 适配）完全不同：不针对具体场景做适配，追求通用泛化能力。仿真数据提供物理规律基础，真机数据仅作后期校正。

团队背景：首席技术顾问苏昊（复旦教授，ImageNet/ShapeNet/PointNet 核心人物），投资方包括宁德时代、阿里、高瓴、腾讯、蚂蚁、IDG 等。已与宁德时代在电池生产场景联合开发。

**核心洞察：** 如果纯仿真路线被验证可行，具身智能的数据瓶颈将被彻底打破——不再需要昂贵的真机数据采集，模型能力可以随算力线性扩展。

## 6. Anthropic 编程智能体负责人的 Vibe Coding 大师课

**来源：** 机器之心  
**关键词：** Vibe Coding、Anthropic、AI 编程

Anthropic 研究员 Erik Schluntz（《构建高效智能体》合著者）分享了在生产环境中负责任地 Vibe Coding 的方法论。他曾因手骨折打石膏两个月，被迫全权交给 Claude 编程，由此积累了极端场景下的实战经验。

核心观点：

- **真正的 Vibe Coding 定义**：不是用 Copilot 逐行审查，而是「忘记代码的存在，只关注产品」
- **为什么要拥抱**：AI 能独立处理的任务长度每 7 个月翻倍，明年 AI 一次生成 1 周工作量的代码时，逐行审查将成为瓶颈
- **叶子节点策略**：只在不被其他模块依赖的末端功能上放手让 AI 写，核心架构仍需人工保护
- **做 AI 的产品经理**：花 15-20 分钟让 Claude 探索代码库、制定计划，再一次性执行，成功率指数级提升
- **22000 行生产代码案例**：Anthropic 内部成功合并 22000 行 Claude 编写的代码，原本 2 周的工作压缩到 1 天

实战技巧：
- TDD 极其有用，但要限制 Claude 只写 3 个极简 E2E 测试（happy path + 2 个错误场景）
- 在「人类程序员会停下来吃午饭」的节点做 Compact，丢掉规划阶段的 10 万 token
- 用 Claude Code 搭框架 + Cursor 收尾修复，面对陌生代码库先探索再动手

**与我们项目的关联：** 这篇跟 #3 CLAUDE.md 是同一主题的两面——#3 是「怎么用规则管住 AI」，这篇是「怎么用工作流释放 AI」。我们的 CLAUDE.md + finish skill 流程本质上就是这套方法论的实践。

## 7. 马斯克官宣 Grok 5 就是 AGI，五月连轰两代万亿模型

**来源：** 新智元  
**关键词：** xAI、AGI、大模型竞赛

马斯克在 X 上直接宣称「Grok 5 就是 AGI」，并甩出路线图：

- Grok 4.3（已上线）→ 0.5T 参数 Beta
- Grok 4.4（5 月初）→ 1T 参数，编程和长上下文飞跃
- Grok 4.5（5 月底）→ 1.5T 参数
- Grok 5（训练中）→ 6T 参数，在 Colossus 2 集群上训练（55 万块 GB200/GB300，2 吉瓦功率）

一个月内从 0.5T 到 1T 到 1.5T 的三级跳，AI 史上从未有过。

xAI 的独特优势：X 平台每天 6800 万条推文的实时数据、特斯拉车队的物理世界传感器数据、SpaceX 级别的工程执行力（122 天建成吉瓦级超算）。

多智能体架构演进：Grok 4.20 的 4 智能体 → Heavy 的 16 智能体 → Grok 5 的动态智能体生成。

**冷静看待：** Karpathy 明确表示 AGI 仍需十年。参数堆叠是否能跨越到 AGI 存在根本性争议。但 xAI 的执行速度和数据优势确实让竞争格局更加紧张。

## 8. OpenAI 内部数据智能体：Codex + 终身记忆，SQL 查询难度归零

**来源：** 新智元  
**关键词：** OpenAI、Agent、记忆系统

OpenAI 内部曝光了一个由 Codex 驱动的数据分析智能体，用六层上下文架构接管了「找表→懂表→写 SQL→校验结果」全链路。

六层架构（从底到顶）：

1. **基础元数据** — 表结构、列类型
2. **人工标注** — 领域专家写的语义描述和注意事项
3. **Codex 增强** — 通过代码推导表的实际内容、更新频率、数据范围
4. **机构知识** — 接入 Slack/Docs/Notion，获取产品发布、事件、指标定义
5. **学习进化（终身记忆）** — 保存纠正经验，支持全局/个人记忆，可手动编辑
6. **运行时上下文** — 实时查询数据仓库，与 Airflow/Spark 通信

工作流：每天凌晨离线扫描数据表用法 → 生成知识图谱 → 向量化存入检索库。用户提问时 RAG 精准定位 + 实时查询通道协同。

踩坑经验：
- 不要给 AI 访问全部数据表，限制范围减少歧义
- 不要写死板的系统提示词，让 AI 自己决定实现方式
- 代码比元数据更能揭示表的真实含义（用 Codex 爬代码库理解数据集构建逻辑）

**与我们项目的关联：** 这个六层记忆架构跟我们的 memory 系统思路一致——分层、可编辑、持久化。特别是第 5 层「学习进化」，本质就是我们的 feedback memory。

## 9. DeepSeek DeepGEMM — 底层算子级开源

**来源：** GitHub Trending  
**关键词：** DeepSeek、开源、GPU 算子

DeepSeek 开源的统一高性能 Tensor Core 内核库，把 LLM 关键计算原语整合到单一 CUDA 代码库：

- FP8 / FP4 / BF16 GEMM（多精度矩阵乘法）
- Mega MoE — 将 EP dispatch → Linear → SwiGLU → Linear → EP combine 融合为单一 mega-kernel，NVLink 通信与计算重叠
- MQA Scoring — DeepSeek v3.2 indexer 的注意力内核
- JIT 编译 — 运行时按需编译，零安装开销

性能：H800 上达到 1550 TFLOPS。设计极简，核心内核函数数量有限，适合学习 GPU 优化。支持 SM90（Hopper）+ SM100（Blackwell）双架构。

**为什么重要：** 这是 Mythos 逆推开源（#2/#4）的底层支撑——OpenMythos 的 MoE 设计借鉴 DeepSeekMoE，而 DeepGEMM 就是跑这些 MoE 层的高性能算子。DeepSeek 在开源生态的布局是从算子到架构的全栈覆盖。

## 今日趋势总结（更新版）

四个信号值得持续关注：

1. **架构创新 > 参数堆叠**：Mythos 循环深度 Transformer（一半参数追平传统模型）、苏度纯仿真路线、DeepGEMM 的算子级优化——都在说暴力 scaling 不是唯一出路。但马斯克的 6T Grok 5 说明参数派还没认输
2. **AI 编程进入「工程化管理」阶段**：CLAUDE.md 定义行为规范、Vibe Coding 大师课定义工作流、Qwen3.6 主攻智能体编程能力——焦点从「AI 能不能写代码」转向「怎么系统性地管好 AI 写代码」
3. **记忆系统成为 Agent 核心差异化**：OpenAI 的六层记忆架构让数据智能体具备终身学习能力，这是从工具到队友的范式转变
4. **开源全栈瓦解闭源护城河**：DeepSeek 从算子（DeepGEMM）到架构（DeepSeekMoE）全开源，Mythos 被逆推复现，千问全系列覆盖——闭源优势窗口期越来越短
